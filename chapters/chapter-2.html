01

JADI GINI CERITANYA... KENAPA SIH JSX ITU ADA? 🤔

Konsep inti · 8 menit baca

Coba bayangin kamu lagi bikin UI pakai JavaScript vanilla. Mau bikin cuma sekadar heading sama paragraph, kode yang kamu tulis udah kayak nerangin resep ke robot 🤖:

Nah, di situlah JSX (JavaScript XML) datang sebagai pahlawan. JSX itu ibarat kamu nulis "HTML di dalam JavaScript" — tapi dengan kekuatan penuh JavaScript di belakangnya. Keren nggak? 😎

JSX — HTML di dalam JavaScript 
// JSX = HTML-like syntax di JavaScript
const element = <h1 className="title">Hello React!</h1>;

// Expression di dalam JSX pakai curly braces
const name = "Arif";
const greeting = <h1>Halo, {name}!</h1>;

// Conditional rendering
const isLoggedIn = true;
const nav = isLoggedIn ? <LogoutButton /> : <LoginButton />;

// List rendering dengan .map()
const fruits = ["Apple", "Banana", "Cherry"];
const list = (
  <ul>
    {fruits.map((fruit, i) => (
      <li key={i}>{fruit}</li>
    ))}
  </ul>
);

Intinya gini: JSX itu bukan HTML. Dia cuma keliatan kayak HTML. Di balik layar, Babel (tool build-nya) bakal nge-transpile semua JSX kamu jadi fungsi React.createElement(). Jadi browser bisa paham. Mirip kayak penerjemah bahasa — kamu ngomong bahasa manusia, dia nerjemahin ke bahasa mesin 🧠

Kenapa nggak langsung pake React.createElement() aja? Jawabannya simpel: coba liat kode di bawah ini. Mana yang mau kamu maintain selama setahun? 👇

Oh iya, satu hal penting — JSX itu mendukung JavaScript expressions di dalam kurung kurawal {}. Artinya kamu bisa masukin variabel, panggil fungsi, bikin kalkulasi, bahkan nulis kondisi langsung di template UI kamu. Serba bisa pokoknya 💪

JSX vs React.createElement() — pilih mana? 😅 
// Cara LAMA: tanpa JSX (mata berdarah 💀)
const element = React.createElement(
  'h1',
  { className: 'greeting' },
  'Halo, ',
  React.createElement('span', null, 'Dunia')
);

// Cara BARU: dengan JSX (hati tenang, pikiran jernih ✅)
const element = (
  <h1 className="greeting">
    Halo, <span>Dunia</span>
  </h1>
);

// Hasilnya SAMA PERSIS di browser!
// JSX itu cuma "pemanis" biar kode kita enak dibaca
// Bayangin kalo kode ribuan baris pake createElement... 🙃

💡 Analogi sederhana: JSX itu kayak kamu pesan makan di warung. Tinggal bilang "nasi goreng telur mata sapi, pedes, nggak pake kerupuk" — done. Sementara React.createElement() itu kayak kamu kasih instruksi detail ke koki: "ambil nasi 200gram, masukkan wajan, nyalakan api sedang..." — sama hasilnya, tapi capek banget ngomongnya 😂

02

ATURAN JSX — KALO NGGAK IKUT, ERROR MENUNGGU 😤

Fundamental · 10 menit baca

OK jadi tadi kamu udah liat betapa enaknya JSX dibanding createElement(). Tapi hati-hati ya — JSX itu punya aturan ketat yang beda dari HTML biasa. Banyak developer pemula yang kena error di sini dan bingung "kok bisa ya padahal di HTML biasa jalan?". Yuk kita bahas satu-satu biar nggak kena jebakan 🪤

ATURAN 1: HARUS ADA SATU ROOT ELEMENT. Setiap komponen React itu harus return tepat satu elemen induk. Nggak boleh return dua elemen sekaligus. Kalau kamu butuh banyak elemen, bungkus pake Fragment (<>...</>). Fragment itu kayak wrapper bayangan — nge-bungkus elemen-elemen kamu tapi nggak nambahin <div> baru di DOM. Clean banget! 🧹

Aturan 1 — Root Element & Fragment 
// Parent kirim props ke child
function App() {
  return (
    <div>
      <Card title="React Basics" author="Arif" views={1200} />
      <Card title="Hooks Guide" author="Budi" views={850} />
    </div>
  );
}

// Child terima props
function Card({ title, author, views }) {
  return (
    <div className="card">
      <h2>{title}</h2>
      <p>By: {author}</p>
      <span>{views} views</span>
    </div>
  );
}

// Children prop untuk komposisi
function Container({ children }) {
  return <div className="p-6 rounded-xl">{children}</div>;
}

ATURAN 2: SEMUA TAG HARUS DITUTUP. Di HTML biasa, lu bisa nulis <img> atau <br> begitu aja tanpa tutup. Di JSX? Lupakan. Setiap tag harus ditutup — pake closing tag atau self-closing slash. <img />, <br />, <input />. Ini salah satu penyebab error paling sering buat orang yang baru migrasi dari HTML 😅

ATURAN 3: camelCase UNTUK ATRIBUT. Ini yang bikin paling banyak orang salah. Karena JSX itu JavaScript, semua nama atribut harus pakai camelCase. Jadi: className (bukan class), htmlFor (bukan for), onClick (bukan onclick). Kebiasaan dari HTML harus dilupain sementara ya 🧠

ATURAN 4: MASUKIN JAVASCRIPT PAKAI KURUNG KURAWAL {}. Mau nampilin variabel? Pakai {}. Mau nulis kondisi? Pakai {}. Mau panggil fungsi? Pakai {}. Tanpa kurung kurawal, JSX nggak akan ngeh itu kode JavaScript — dia anggap cuma text biasa 📝

Bener vs Salah — dont be like the wrong side 😂 
import { useState } from 'react';

function ProfilPengguna({ nama, umur, isOnline }) {
  // ✅ ATURAN 1: Satu root element (menggunakan Fragment)
  return (
    <>
      {isOnline && <span className="badge">● Online</span>}
      
      {/* ✅ Aturan 2: Semua tag ditutup */}
      <img src="/avatar.jpg" alt="Avatar" />
      
      {/* ✅ Aturan 3: camelCase untuk atribut */}
      <h2 className="text-xl font-bold">{nama}</h2>
      <label htmlFor="umur">Umur:</label>
      
      {/* ✅ Aturan 4: Ekspresi JS dalam {} */}
      <p>Umur: {umur} tahun (lahir ~{2026 - umur})</p>
      
      <button onClick={() => alert(`Halo ${nama}!`)}>
        Klik
      </button>
      
      {/* Ternary operator juga gaspol */}
      <p>Status: {umur >= 17 ? 'Dewasa' : 'Masih bocil'}</p>
    </>
  );
}

// ❌ JANGAN LAKUKAN INI (auto-kena error):
// return <div><p>A</p><p>B</p>  // Gak ada root!
// return <img src="a.jpg">                  // Tag gak ditutup!
// return <div class="box">                 // Harus className!

⚠️ FYI aja nih: Komentar di JSX itu beda lho dari JavaScript biasa. Kamu nggak bisa pake // di dalam JSX. Harus pake {/* komentar kamu di sini */}. Kalau pake <!-- --> (komentar HTML), teksnya bakal ke-render sebagai tulisan di halaman. Pantesan aja halaman kamu aneh ternyata 😜

03

SI SUPERPOWER KURUNG KURAWAL {} 💎

Praktik · 8 menit

Ini bagian yang paling seru dari JSX. Kurung kurawal {} itu kayak "portal" buat masukin kekuatan penuh JavaScript ke dalam template UI kamu. Seriusan, kamu bisa ngapain aja di dalam situ — variabel, fungsi, kalkulasi, kondisi, sampe render array sekaligus 🚀

Nih daftar yang BOLEH kamu masukin ke dalam {}: Variabel, string, angka, boolean — tampilin data dari state atau props. Ekspresi matematika — kayak {harga * jumlah}. Panggilan fungsi — bikin helper function terus panggil di JSX. Ternary operator — buat kondisi singkat kayak {isLogin ? 'Logout' : 'Login'}. Array — React bakal otomatis render tiap item-nya satu-satu!

Ekspresi JS di dalam Kurung Kurawal 
// useState = hook untuk state
function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>+1</button>
      <button onClick={() => setCount(0)}>Reset</button>
    </div>
  );
}

// Form dengan multiple state
function LoginForm() {
  const [email, setEmail] = useState('');
  const [password, setPassword] = useState('');

  return (
    <form onSubmit={e => { e.preventDefault(); console.log(email, password); }}>
      <input value={email} onChange={e => setEmail(e.target.value)} />
      <input type="password" value={password}
        onChange={e => setPassword(e.target.value)} />
      <button type="submit">Login</button>
    </form>
  );
}

Sedangkan yang GAK BOLEH: pernyataan if/else (pake ternary aja), perulangan for/while (pake .map()), dan deklarasi var/let/const (definisikan di atas JSX). Kenapa? Karena di dalam {} itu cuma boleh expression (yang ngasilin nilai), bukan statement (yang menjalankan aksi) 🎓

Kalau kondisinya udah terlalu kompleks buat ternary, triknya gampang: hitung dulu di luar JSX, baru masukin hasilnya sebagai variabel. Kode jadi rapi dan nggak pusing dibaca. Check contohnya di bawah ini 👇

Bukti kehebatan kurung kurawal 🔥 
import { useState } from 'react';

function DaftarBelanja() {
  const [items, setItems] = useState([
    { nama: 'Beras', harga: 15000, jumlah: 2 },
    { nama: 'Telur', harga: 28000, jumlah: 1 },
    { nama: 'Minyak', harga: 18000, jumlah: 1 },
  ]);
  const [diskon, setDiskon] = useState(10);

  // Hitung dulu di LUAR JSX — biar gak ribet
  const totalHarga = items.reduce((sum, item) => 
    sum + (item.harga * item.jumlah), 0
  );
  const hargaDiskon = totalHarga * (1 - diskon / 100);

  // Fungsi helper
  const formatRupiah = (angka) => {
    return new Intl.NumberFormat('id-ID', {
      style: 'currency', currency: 'IDR',
      minimumFractionDigits: 0
    }).format(angka);
  };

  return (
    <div className="p-6">
      <h2 className="text-2xl font-bold mb-4">🛒 Daftar Belanja</h2>
      
      {/* Render array pake .map() — React auto-render tiap item */}
      {items.map((item, index) => (
        <div key={index} className="flex justify-between p-3 border-b">
          <span>{item.nama} × {item.jumlah}</span>
          {/* Ekspresi matematika di dalam {} */}
          <span>{formatRupiah(item.harga * item.jumlah)}</span>
        </div>
      ))}
      
      <div className="mt-4 p-4 bg-gray-800 rounded-lg">
        <p>Total: {formatRupiah(totalHarga)}</p>
        <p>Diskon ({diskon}%): -{formatRupiah(totalHarga * diskon / 100)}</p>
        <p className="text-xl font-bold text-green-400">
          Bayar: {formatRupiah(hargaDiskon)}
        </p>
        
        {/* Ternary operator — kondisi dalam satu baris */}
        {totalHarga > 50000 ? (
          <p className="text-amber-600">🎉 Borongan nih! Dapet cashback!</p>
        ) : (
          <p className="text-muted">Tambah lagi yuk!</p>
        )}
      </div>
    </div>
  );
}

export default DaftarBelanja;

💡 Pro tip: Mau format angka jadi Rupiah? Jangan pake manual string concatenation (udah kuno 😅). Pakai Intl.NumberFormat('id-ID') — fitur bawaan JavaScript yang gratis, support banyak mata uang, dan nggak perlu install library apapun. Hidden gem banget ini!

04

GAYA-GAYAAN DI JSX — BIAR GAK CUPU 🎨

Praktik · 9 menit

UI tanpa style itu kayak nasi goreng tanpa kecap — masalah sih masih bisa dimakan, tapi nggak ada yang mau liat 👀. Di React (dan JSX tentunya), ada beberapa cara buat kasih style. Masing-masing punya momen dimana dia paling cocok dipake.

Metode 1: Inline Styles. Ini paling gampang — langsung tulis style sebagai objek JavaScript di properti style. Tapi hati-hati, karena ini JavaScript, nama properti CSS-nya harus camelCase. Jadi background-color jadi backgroundColor, font-size jadi fontSize. Angka buat unit px nggak perlu dikasih tanda petik — React otomatis tambahin. Cocok buat style yang dinamis (bergantung state) 🎯

Metode 2: CSS Modules. Ini pilihan yang paling seimbang menurut banyak developer. Kamu tulis CSS biasa di file terpisah (nama filenya akhiran .module.css), terus import sebagai objek JavaScript. Kelebihannya: nama kelas CSS-nya otomatis jadi unik saat build, jadi nggak akan ada konflik antar komponen. Kayak punya kamar mandi pribadi — nggak perlu antri 😂

Metode 3: Conditional Class Names. Teknik ini sering dipake sama Tailwind CSS user (kayak yang di proyek ini!). Intinya: tentukan kelas CSS berdasarkan kondisi, pake template literal atau ternary. Misalnya tombol aktif punya warna hijau, kalau nggak aktif warna abu-abu. Simple tapi efektif! 💅

3 teknik styling — pick your fighter! ⚔️ 
// ====== CARA 1: Inline Styles (langsung di tag) ======
function KartuInline() {
  const [hovered, setHovered] = useState(false);
  
  
  // Style itu OBJEK JavaScript, bukan string!
  const cardStyle = {
    padding: '1.5rem',
    borderRadius: '1rem',
    backgroundColor: hovered ? '#E5E5E5' : '#F5F4F1',
    border: '1px solid #D1D1D1',
    // angka otomatis ditambah 'px'
    fontSize: 16,  
    color: '#333333',
    transition: 'background-color 0.3s ease',
    boxShadow: hovered 
      ? '0 8px 30px rgba(39,216,144,0.15)' 
      : 'none',
  };

  return (
    <div 
      style={cardStyle}
      onMouseEnter={() => setHovered(true)}
      onMouseLeave={() => setHovered(false)}
    >
      <h3 style={{ color: '#0B0B0B', marginBottom: '0.5rem' }}>
        Kartu Inline Style
      </h3>
      <p>Style berubah pas di-hover! Coba aja 👆</p>
    </div>
  );
}

// ====== CARA 2: CSS Modules (paling aman >) ======

// .judul { color: #0B0B0B; margin-bottom: 0.5rem; }

// ====== CARA 3: Conditional Class Names (buat logic dinamis) ======
function TombolStatus({ isActive }) {
  return (
    <button className={`px-4 py-2 rounded-lg font-bold transition-colors
      ${isActive 
        ? 'bg-green-500 text-ink' 
        : 'bg-gray-700 text-muted'}

}

⚠️ Jebakan paling sering: Lupa kalau properti CSS di inline style itu wajib camelCase. Nulis background-color (pake dash) bakal langsung error karena JavaScript nggak ngerti dash di nama properti. Error message-nya juga kadang nggak jelas. Jadi hafalin ini ya: backgroundColor, fontSize, borderRadius. Semua disambung, tanpa dash! 🔗

PROPS — KIRIM DATA ANTAR KOMPONEN 📦

Fundamental · 10 menit baca

Bayangin kamu lagi pesan ojek online. Kamu kasih tahu driver: "jemput di rumah, tujuan kantor, lewat jalan tol". Driver nggak perlu tau kenapa kamu ke kantor, driver cuma perlu tau data yang kamu kasih. Nah, di React, konsep ini namanya Props (Properties). Dia itu kayak data yang kamu lempar dari komponen induk ke komponen anak 🏍️

Props itu sifatnya read-only. Komponen anak nggak boleh mengubah props yang dikasih induk. Ini aturan fundamental React — kalau kamu melanggar ini, aplikasi kamu bakal jadi unpredictable. Mikirin aja kayak driver ojol nggak bisa tiba-tiba ngubah tujuan kamu tanpa kamu bilang 😅

Berikut cara-cara mengirim dan menerima props:

Props Basics — kirim & terima data 
// ===== CARA 1: Props biasa =====
function Card({ title, author }) {
  return (
    <div className="bg-white rounded-xl p-4 shadow-sm">
      <h3 className="font-bold">{title}</h3>
      <p className="text-gray-500">Oleh: {author}</p>
    </div>
  );
}

// Kirim dari parent
function App() {
  return (
    <div>
      <Card title="Belajar React" author="Arif" />
      <Card title="TypeScript Mastery" author="Budi" />
    </div>
  );
}

// ===== CARA 2: Default Props =====
function Tombol({ label = "Klik Saya", variant = "primary" }) {
  const bgColor = variant === "primary"
    ? "bg-black text-white"
    : "bg-gray-100 text-gray-800";

  return (
    <button className={`px-4 py-2 rounded-lg ${bgColor}`}>
      {label}
    </button>
  );
}

// Kalo nggak kirim props, pake default
<Tombol />                        // "Klik Saya", primary
<Tombol label="Hapus" variant="danger" />  // "Hapus", danger

Sekarang kita bahas children prop — ini props spesial yang otomatis tersedia di setiap komponen. Children itu berisi semua elemen yang kamu taruh di antara opening dan closing tag komponen. Super berguna buat bikin komponen wrapper yang fleksibel!

Children Prop — wrapper component fleksibel 
// Children = semua yang ada di antara tag pembuka & penutup
function Modal({ isOpen, title, children }) {
  if (!isOpen) return null;

  return (
    <div className="fixed inset-0 bg-black/50 flex items-center justify-center">
      <div className="bg-white rounded-2xl p-6 max-w-md w-full mx-4">
        <h2 className="text-xl font-bold mb-4">{title}</h2>
        {/* children = konten apapun dari parent */}
        {children}
        <button className="mt-4 px-4 py-2 bg-gray-100 rounded-lg">
          Tutup
        </button>
      </div>
    </div>
  );
}

// Pakai modal buat berbagai keperluan — fleksibel!
function App() {
  return (
    <div>
      {/* Modal konfirmasi hapus */}
      <Modal isOpen={true} title="Konfirmasi">
        <p>Yakin mau hapus data ini? Aksi ini nggak bisa di-undo.</p>
      </Modal>

      {/* Modal form — PAKAI KOMPONEN YANG SAMA! */}
      <Modal isOpen={false} title="Edit Profil">
        <input placeholder="Nama" className="border p-2 rounded-lg w-full" />
        <input placeholder="Email" className="border p-2 rounded-lg w-full mt-2" />
      </Modal>
    </div>
  );
}

Satu konsep penting yang perlu kamu tau: Prop Drilling. Kalau data perlu sampai ke komponen yang sangat dalam (misal App → Navbar → UserMenu → Avatar), kamu harus lempar props dari setiap level. Ini bisa jadi ribet kalau komponennya banyak. Nanti di Bab 8 (State Management) kita bakal bahas cara mengatasinya. Buat sekarang, prop drilling untuk 2-3 level masih oke kok 👍

💡 Pro tip: Selalu destructure props di parameter fungsi — function Card({ title, author }) — daripada akses lewat props.title. Kode jadi lebih ringkas, dan kamu langsung keliatan prop apa aja yang dipakai komponen itu. Standar industri modern!

06

STATE — DATA YANG BISA BERUBAH 🔄

Fundamental · 12 menit baca

Kalau props itu data statis yang dikasih dari atas (read-only), maka state adalah data internal komponen yang bisa berubah sendiri. Bayangin props itu kayak CD musik (isi-nya tetap, kamu cuma putar), sementara state itu kayak Spotify playlist — kamu bisa tambah, hapus, shuffle kapan aja 🎵

Di React modern, state dikelola lewat hook useState(). Hook ini mengembalikan dua hal: nilai state saat ini dan fungsi buat mengubahnya. Setiap kali state berubah, React otomatis re-render komponen itu (dan semua anak-anaknya) dengan data terbaru. Ini konsep yang bikin React "reactive"!

useState — dasar-dasar state management 
import { useState } from 'react';

// State dengan nilai awal sederhana
function LikeButton() {
  const [likes, setLikes] = useState(0);

  return (
    <button onClick={() => setLikes(likes + 1)}>
      ❤️ {likes} suka
    </button>
  );
}

// State dengan objek — WAJIB SPREAD!
function FormPendaftaran() {
  const [form, setForm] = useState({
    nama: '',
    email: '',
    jurusan: 'Informatika'
  });

  const handleChange = (e) => {
    const { name, value } = e.target;
    // SPREAD existing state, lalu update field-nya
    setForm((prev) => ({ ...prev, [name]: value }));
  };

  return (
    <form>
      <input
        name="nama"
        value={form.nama}
        onChange={handleChange}
        placeholder="Nama kamu"
      />
      <input
        name="email"
        value={form.email}
        onChange={handleChange}
        placeholder="Email kamu"
      />
      <select name="jurusan" value={form.jurusan}
        onChange={handleChange}>
        <option value="Informatika">Informatika</option>
        <option value="Desain">Desain Komunikasi Visual</option>
        <option value="Manajemen">Manajemen</option>
      </select>
      <p>Nama: {form.nama}, Email: {form.email}, Jurusan: {form.jurusan}</p>
    </form>
  );
}

Aturan Emas #1: State itu IMMUTABLE. Jangan pernah mengubah state langsung. Kalau state kamu objek atau array, selalu buat copy baru sebelum mengubah. Kalau langsung mutasi, React nggak bakal ngeh kalau ada perubahan — dan UI-nya nggak akan update! Ini salah satu penyebab bug paling bikin pusing di React 😤

Aturan Emas #2: Pake Updater Function untuk state yang bergantung pada state sebelumnya. Kalau kamu butuh nilai state lama buat ngitung nilai baru (misalnya counter +1), jangan tulis setCount(count + 1) — tapi tulis setCount(prev => prev + 1). Kenapa? Karena React bisa nge-batch update state, dan nilai count mungkin udah "stale" pas eksekusi 🧠

Updater Function & Derived State 
import { useState } from 'react';

// ❌ SALAH — state-nya bisa stuck karena stale closure
function BrokenCounter() {
  const [count, setCount] = useState(0);

  const addThreeWrong = () => {
    setCount(count + 1); // pakai count yang sama 3x!
    setCount(count + 1); // hasilnya cuma +1, bukan +3
    setCount(count + 1);
  };

  // ✅ BENAR — updater function selalu pakai nilai terbaru
  const addThreeRight = () => {
    setCount(prev => prev + 1); // chain update
    setCount(prev => prev + 1); // masing-masing pakai latest value
    setCount(prev => prev + 1); // hasilnya: +3 ✅
  };

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={addThreeRight}>+3 (aman)</button>
      <button onClick={() => setCount(0)}>Reset</button>
    </div>
  );
}

// Derived State — nggak perlu state terpisah!
function ProductCard({ price, taxRate = 0.11 }) {
  // ❌ JANGAN: simpan total di state terpisah
  // const [total, setTotal] = useState(price * (1 + taxRate));

  // ✅ BENAR: hitung langsung dari state/props yang ada
  const total = price * (1 + taxRate);
  const formatted = new Intl.NumberFormat('id-ID', {
    style: 'currency', currency: 'IDR',
  }).format(total);

  return <p>Total: {formatted} (termasuk PPN)</p>;
}

Aturan Emas #3: Derived state nggak perlu state terpisah. Kalau suatu nilai bisa dihitung dari state atau props lain, jangan simpan dia di state sendiri. Contoh: total harga bisa dihitung dari harga satuan × jumlah. Simpen di state justru bikin data inkonsisten (harga berubah tapi total lupa di-update). Hitung aja langsung di render!

⚠️ Kesalahan Pemula Paling Sering: Lupa bahwa setState itu asynchronous — nilai state nggak langsung berubah di baris berikutnya. Kalau kamu nulis setCount(5); console.log(count); — yang ke-log masih nilai lama, bukan 5. Ini bukan bug, ini cara React bekerja. Butuh nilai baru? Pakai useEffect (bahas di Bab 3)!

EVENT HANDLING — REAKSI KETIKA USER AKSI 💥

Praktik · 10 menit baca

Aplikasi tanpa event handling itu kayak restoran tanpa pelayan — makanan ada, tapi nggak ada yang nganterin ke meja kamu 😂. Event handling itu cara kita bikin aplikasi React merespons aksi user: klik tombol, ketik di input, submit form, hover mouse, scroll halaman, dan lain-lain.

Di React, event handling mirip dengan HTML biasa tapi ada 3 perbedaan penting: pertama, nama eventnya pakai camelCase (onClick bukan onclick). Kedua, kamu kasih fungsi sebagai handler, bukan string. Ketiga, React pakai Synthetic Events — wrapper buatan React yang konsisten di semua browser 🌐

Event Handlers — onClick, onChange, onSubmit 
import { useState } from 'react';

// onClick — klik tombol
function TombolCetak() {
  const [log, setLog] = useState([]);

  const handleClick = () => {
    setLog(prev => [...prev, `Klik terakhir: ${new Date().toLocaleTimeString('id-ID')}`]);
  };

  return (
    <div>
      <button onClick={handleClick}>Cetak Log</button>
      {log.map((entry, i) => <p key={i}>{entry}</p>)}
    </div>
  );
}

// onChange — input berubah
function Pencarian() {
  const [query, setQuery] = useState('');

  return (
    <div>
      <input
        type="text"
        value={query}
        onChange={(e) => setQuery(e.target.value)}
        placeholder="Cari sesuatu..."
      />
      <p>Kamu mengetik: {query || '...'}</p>
    </div>
  );
}

// onSubmit — form submission
function FormLogin() {
  const [email, setEmail] = useState('');
  const [loading, setLoading] = useState(false);

  const handleSubmit = async (e) => {
    e.preventDefault(); // WAJIB! cegah page reload
    setLoading(true);

    // Simulasi API call
    setTimeout(() => {
      console.log('Login dengan:', email);
      setLoading(false);
    }, 1500);
  };

  return (
    <form onSubmit={handleSubmit}>
      <input
        type="email"
        value={email}
        onChange={(e) => setEmail(e.target.value)}
        placeholder="email@contoh.com"
        required
      />
      <button type="submit" disabled={loading}>
        {loading ? 'Memproses...' : 'Login'}
      </button>
    </form>
  );
}

Sintaks inline vs Handler Terpisah: Kamu bisa tulis handler langsung di JSX (onClick={() => setX(true)}) atau pisahkan ke fungsi terpisah (onClick={handleClick}). Untuk handler yang cuma 1 baris, inline itu oke. Tapi kalau logic-nya lebih dari 2 baris, PISAHKAN. Kode jadi rapi dan bisa di-test 👌

Synthetic Events: React membungkus event browser native dengan SyntheticEvent — objek yang punya API sama persis dengan event native tapi konsisten di semua browser (Chrome, Firefox, Safari). Jadi kamu nggak perlu khawatir tentang perbedaan implementasi event antar browser 🎯

Event Object yang berguna: Setiap handler menerima objek e (event) yang berisi informasi penting: e.target (elemen yang trigger event), e.target.value (nilai input), e.preventDefault() (cegah default behavior), e.stopPropagation() (hentikan event naik ke parent).

⚠️ Jangan lupa preventDefault!: Kalau kamu punya form dan handler onSubmit, SELALU panggil e.preventDefault() di baris pertama handler. Kalau nggak, browser bakal nge-reload halaman setiap kali form disubmit — dan semua state kamu hilang! Ini error paling sering buat pemula yang bikin form di React.

Langkah 1: Layout — Pertama, kita bikin struktur UI kalkulator. Di sini kita pake display buat nampilin angka, dan grid tombol buat angka 0-9 plus operator (+, -, ×, ÷, =, C).

Step 1 — Layout Kalkulator 
import { useState } from 'react';

function Kalkulator() {
  const [display, setDisplay] = useState('0');
  const [previousValue, setPreviousValue] = useState(null);

    { label: '-', type: 'operator', value: '-' },
    { label: '1', type: 'number' },
    { label: '2', type: 'number' },
    { label: '3', type: 'number' },
    { label: '+', type: 'operator', value: '+' },
    { label: '0', type: 'number' },

Langkah 2: Logic — Sekarang kita implementasikan fungsi handleButton() dan calculate() yang jadi otak kalkulator.

Step 2 — Logic Kalkulator 
// Tambahkan ini di dalam komponen Kalkulator (di atas return)

const calculate = (a, b, op) => {
  switch (op) {
    case '+': return a + b;
    case '-': return a - b;
    case '*': return a * b;
    case '/': return b !== 0 ? a / b : 'Error';
    default: return b;
  }
};

const handleButton = (btn) => {
  switch (btn.type) {
    case 'number':
      if (waitingForOperand) {
        setDisplay(btn.label);
        setWaitingForOperand(false);
      } else {
        setDisplay(display === '0' ? btn.label : display + btn.label);
      }
      break;

    case 'operator':
      const current = parseFloat(display);
      if (previousValue !== null && !waitingForOperand) {
        const result = calculate(previousValue, current, operator);
        setDisplay(String(result));
        setPreviousValue(result);
      } else {
        setPreviousValue(current);
      }
      setOperator(btn.value);
      setWaitingForOperand(true);
      break;

    case 'equals':
      if (previousValue !== null && operator) {
        const current = parseFloat(display);
        const result = calculate(previousValue, current, operator);
        setDisplay(String(result));
        setPreviousValue(null);
        setOperator(null);
        setWaitingForOperand(true);
      }
      break;

    case 'clear':
      setDisplay('0');
      setPreviousValue(null);
      setOperator(null);
      setWaitingForOperand(false);
      break;

    case 'decimal':
      if (!display.includes('.')) {
        setDisplay(display + '.');
      }
      break;

    case 'negate':
      setDisplay(String(parseFloat(display) * -1));
      break;

    case 'percent':
      setDisplay(String(parseFloat(display) / 100));
      break;
  }
};

export default Kalkulator;
