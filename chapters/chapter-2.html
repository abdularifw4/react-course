01

Memahami JSX

Konsep inti · 8 menit baca

JSX (JavaScript XML) adalah ekstensi sintaks JavaScript yang memungkinkan kamu menulis elemen-elemen UI langsung di dalam kode JavaScript. JSX bukanlah HTML biasa — meskipun terlihat sangat mirip, JSX memiliki aturan dan perilaku khusus yang berbeda dari HTML. Di balik layar, JSX akan dikompilasi menjadi fungsi React.createElement() oleh Babel.

Keuntungan utama JSX adalah kamu bisa menulis struktur UI yang kompleks dengan cara yang intuitif dan mudah dibaca. Daripada menulis fungsi pemanggilan yang bertumpuk, JSX memungkinkan kamu menulis "template HTML" yang sebenarnya adalah kode JavaScript murni. Ini membuat developer bisa memahami struktur komponen hanya dengan melihat kodenya.

Perlu diingat bahwa JSX memiliki beberapa aturan penting: (1) Selalu kembalikan satu elemen root tunggal — jika ada beberapa elemen, bungkus dengan fragment atau <div>. (2) Gunakan camelCase untuk atribut — className bukan class, htmlFor bukan for. (3) Tag self-closing memerlukan slash di akhir: <img />.

JSX juga mendukung JavaScript expressions di dalam kurung kurawal {}. Kamu bisa memasukkan variabel, panggilan fungsi, operasi matematika, ternary operator, dan bahkan render komponen lain di dalam JSX. Ini membuat JSX sangat fleksibel dan powerful.

JSX vs React.createElement() 
// Cara LAMA: tanpa JSX (sulit dibaca)
const element = React.createElement(
  'h1',
  { className: 'greeting' },
  'Halo, ',
  React.createElement('span', null, 'Dunia')
);

// Cara BARU: dengan JSX (mudah dibaca ✅)
const element = (
  <h1 className="greeting">
    Halo, <span>Dunia</span>
  </h1>
);

// Keduanya menghasilkan output yang SAMA persis!
// JSX hanya "syntax sugar" di atas React.createElement()

💡 Analogi: JSX itu seperti menulis resep masakan dalam bahasa Indonesia, sementara React.createElement() adalah instruksi mesin yang kompleks. Keduanya menghasilkan makanan yang sama, tapi JSX jauh lebih mudah dibaca oleh manusia!

02

Aturan-Aturan JSX

Fundamental · 10 menit baca

Menguasai JSX berarti memahami aturan-aturannya. Meskipun JSX mirip HTML, ada perbedaan penting yang jika diabaikan akan menyebabkan error. Berikut adalah aturan-aturan kunci yang harus selalu kamu ingat saat menulis JSX.

Aturan 1: Satu Root Element. Setiap komponen React harus mengembalikan tepat satu elemen induk. Jika kamu perlu mengembalikan beberapa elemen, gunakan Fragment (<>...</>) agar tidak menambahkan node DOM ekstra.

Aturan 2: Semua Tag Harus Ditutup. Di HTML, beberapa tag seperti <img> atau <br> tidak perlu ditutup. Tapi di JSX, semua tag harus ditutup — baik dengan closing tag maupun self-closing slash.

Aturan 3: camelCase untuk Atribut. Karena JSX adalah JavaScript, semua atribut mengikuti konvensi JavaScript. Gunakan className (bukan class), htmlFor (bukan for), onClick (bukan onclick), dan tabIndex (bukan tabindex).

Aturan 4: Ekspresi JavaScript dalam Kurung Kurawal. Untuk menyisipkan variabel, fungsi, atau operasi JavaScript ke dalam JSX, bungkus dengan {}. Tanpa kurung kurawal, JSX akan memperlakukannya sebagai string biasa.

Contoh Aturan JSX yang Benar dan Salah 
import { useState } from 'react';

function ProfilPengguna({ nama, umur, isOnline }) {
  // ✅ Aturan 1: Satu root element (menggunakan Fragment)
  return (
    <>
      {isOnline && <span className="badge">● Online</span>}
      
      {/* ✅ Aturan 2: Semua tag ditutup */}
      <img src="/avatar.jpg" alt="Avatar" />
      
      {/* ✅ Aturan 3: camelCase untuk atribut */}
      <h2 className="text-xl font-bold">{nama}</h2>
      <label htmlFor="umur">Umur:</label>
      
      {/* ✅ Aturan 4: Ekspresi JS dalam {} */}
      <p>Umur: {umur} tahun (lahir ~{2026 - umur})</p>
      
      <button onClick={() => alert(`Halo ${nama}!`)}>
        Klik
      </button>
      
      {/* Ternary operator juga berfungsi */}
      <p>Status: {umur >= 17 ? 'Dewasa' : 'Belum dewasa'}</p>
    </>
  );
}

// ❌ KESALAHAN UMUM:
// return <div><p>A</p><p>B</p>  // Tidak ada root element!
// return <img src="a.jpg">                  // Tag tidak ditutup!
// return <div class="box">                 // Harus className!
// return <div>{nama}</div> tanpa import   // Variabel harus didefinisikan

⚠️ Perhatian: Komentar di JSX menggunakan sintaks {/* komentar */}, BUKAN // atau <!-- -->. Komentar HTML biasa akan ditampilkan sebagai teks di halaman!

03

Menyisipkan Ekspresi & Variabel

Praktik · 8 menit

Salah satu fitur paling powerful dari JSX adalah kemampuan menyisipkan ekspresi JavaScript langsung di dalam template UI. Dengan kurung kurawal {}, kamu bisa menampilkan variabel, memanggil fungsi, melakukan operasi matematika, dan bahkan menulis kondisi logika.

Yang BISA kamu masukkan dalam kurung kurawal: variabel, angka, string, boolean (meskipun boolean tidak ditampilkan), ekspresi matematika, pemanggilan fungsi, ternary operator, array (yang akan dirender item per item), dan komponen React lainnya. Yang TIDAK BISA: pernyataan if/else, perulangan for, dan var/let/const.

Untuk kondisi yang lebih kompleks dari ternary, gunakan pendekatan "render di luar JSX" — hitung hasilnya terlebih dahulu dalam variabel, lalu masukkan variabel tersebut ke dalam JSX. Ini membuat kode lebih bersih dan mudah dibaca.

Beragam Cara Menyisipkan Ekspresi di JSX 
import { useState } from 'react';

function DaftarBelanja() {
  const [items, setItems] = useState([
    { nama: 'Beras', harga: 15000, jumlah: 2 },
    { nama: 'Telur', harga: 28000, jumlah: 1 },
    { nama: 'Minyak', harga: 18000, jumlah: 1 },
  ]);
  const [diskon, setDiskon] = useState(10);

  // Hitung di luar JSX untuk kondisi kompleks
  const totalHarga = items.reduce((sum, item) => 
    sum + (item.harga * item.jumlah), 0
  );
  const hargaDiskon = totalHarga * (1 - diskon / 100);

  // Fungsi helper
  const formatRupiah = (angka) => {
    return new Intl.NumberFormat('id-ID', {
      style: 'currency', currency: 'IDR',
      minimumFractionDigits: 0
    }).format(angka);
  };

  return (
    <div className="p-6">
      <h2 className="text-2xl font-bold mb-4">🛒 Daftar Belanja</h2>
      
      {/* Render array dengan .map() */}
      {items.map((item, index) => (
        <div key={index} className="flex justify-between p-3 border-b">
          <span>{item.nama} × {item.jumlah}</span>
          {/* Ekspresi matematika */}
          <span>{formatRupiah(item.harga * item.jumlah)}</span>
        </div>
      ))}
      
      <div className="mt-4 p-4 bg-gray-800 rounded-lg">
        <p>Total: {formatRupiah(totalHarga)}</p>
        <p>Diskon ({diskon}%): -{formatRupiah(totalHarga * diskon / 100)}</p>
        <p className="text-xl font-bold text-green-400">
          Bayar: {formatRupiah(hargaDiskon)}
        </p>
        
        {/* Ternary operator untuk kondisi */}
        {totalHarga > 50000 ? (
          <p className="text-yellow-400">🎉 Belanja di atas 50rb!</p>
        ) : (
          <p className="text-gray-400">Tambah lagi yuk!</p>
        )}
      </div>
    </div>
  );
}

export default DaftarBelanja;

💡 Tips: Gunakan Intl.NumberFormat untuk memformat angka menjadi mata uang Rupiah. Ini fitur bawaan JavaScript yang sangat berguna dan mendukung berbagai mata uang internasional!

04

Styling di JSX

Praktik · 9 menit

Ada tiga cara utama untuk memberi style pada elemen JSX: Inline Styles (objek JavaScript), CSS Modules (file CSS terpisah yang di-scope otomatis), dan CSS-in-JS Libraries (styled-components, Emotion). Masing-masing memiliki kelebihan untuk use case yang berbeda.

Untuk inline styles di JSX, kamu menggunakan objek JavaScript (bukan string CSS). Nama properti menggunakan camelCase: backgroundColor bukan background-color, fontSize bukan font-size. Nilai harus berupa string, kecuali untuk angka (yang akan otomatis ditambahkan px).

CSS Modules adalah pilihan populer untuk proyek skala menengah hingga besar. Dengan CSS Modules, nama kelas CSS secara otomatis di-generate menjadi nama unik saat build, sehingga tidak ada konflik kelas antar komponen. File CSS diberi nama dengan akhiran .module.css dan di-import sebagai objek JavaScript.

Tiga Cara Styling di React 
// ====== CARA 1: Inline Styles ======
function KartuInline() {
  const [hovered, setHovered] = useState(false);
  
  const cardStyle = {
    padding: '1.5rem',
    borderRadius: '1rem',
    backgroundColor: hovered ? '#1a2240' : '#111827',
    border: '1px solid #243056',
    // angka otomatis ditambah 'px'
    fontSize: 16,  
    color: '#f3f4f6',
    transition: 'background-color 0.3s ease',
    boxShadow: hovered 
      ? '0 8px 30px rgba(39,216,144,0.15)' 
      : 'none',
  };

  return (
    <div 
      style={cardStyle}
      onMouseEnter={() => setHovered(true)}
      onMouseLeave={() => setHovered(false)}
    >
      <h3 style={{ color: '#27d890', marginBottom: '0.5rem' }}>
        Kartu Inline Style
      </h3>
      <p>Style ditulis sebagai objek JavaScript.</p>
    </div>
  );
}

// ====== CARA 2: CSS Modules ======
// File: Kartu.module.css
// .kartu { padding: 1.5rem; border-radius: 1rem; }
// .judul { color: #27d890; margin-bottom: 0.5rem; }

// import styles from './Kartu.module.css';
function KartuModule() {
  return (
    <div className={styles.kartu}>
      <h3 className={styles.judul}>Kartu CSS Module</h3>
      <p>Kelas CSS otomatis unik, tidak ada konflik!</p>
    </div>
  );
}

// ====== CARA 3: Conditional Class Names ======
function TombolStatus({ isActive }) {
  return (
    <button className={`px-4 py-2 rounded-lg font-bold transition-colors
      ${isActive 
        ? 'bg-green-500 text-white' 
        : 'bg-gray-700 text-gray-400'}
    `}>
      {isActive ? 'Aktif ✓' : 'Nonaktif'}
    </button>
  );
}

⚠️ Kesalahan Umum: Jangan lupa bahwa di inline style, properti CSS menggunakan camelCase. Menulis background-color (dengan dash) akan menyebabkan error karena dash tidak valid dalam nama properti JavaScript. Gunakan backgroundColor sebagai gantinya.
