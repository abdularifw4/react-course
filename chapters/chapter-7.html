Next.js Fundamentals

1. App Router — Routing Modern Next.js

App Router (Next.js 13+) menggunakan folder-based routing. Setiap folder di app/ menjadi route. file page.tsx = halaman, layout.tsx = layout bersama, loading.tsx = loading state otomatis.

// Struktur folder = routing
// app/
//   layout.tsx          -> Root layout (HTML, body)
//   page.tsx             -> /
//   about/
//     page.tsx           -> /about
//   blog/
//     page.tsx           -> /blog
//     [slug]/
//       page.tsx         -> /blog/:slug (dynamic route)
//   dashboard/
//     layout.tsx         -> Dashboard layout (nested)
//     page.tsx           -> /dashboard
//     settings/
//       page.tsx         -> /dashboard/settings

// Dynamic route
interface BlogPostProps {
  params: { slug: string };
}

export default function BlogPost({ params }: BlogPostProps) {
  return <h1>Blog Post: {params.slug}</h1>;
}

// Generate static params untuk SSG
export async function generateStaticParams() {
  const posts = await fetch('https://api.example.com/posts');
  return (await posts.json()).map((p: any) => ({
    slug: p.slug
  }));
}

💡 Tip: App Router itu seperti peta rumah. Setiap kamar (folder) punya pintu sendiri (route). Kamu nggak perlu konfigurasi router — cukup bikin folder dan file!

2. Server Components vs Client Components

React Server Components (RSC) render di server, bukan di browser. Ini mengurangi JavaScript yang dikirim ke client dan memungkinkan akses langsung ke database, file system, dll.

// Server Component (default di Next.js App Router)
// Tidak ada 'use client' -> ini server component
async function ProductList() {
  // Bisa langsung akses database!
  const products = await db.product.findMany();

  return (
    <div>
      {products.map(p => (
        <div key={p.id}>{p.name} - Rp{p.price}</div>
      ))}
    </div>
  );
}

// Client Component — butuh interaktivitas
'use client'; // WAJIB untuk event handlers, hooks, browser API

import { useState } from 'react';

function AddToCart({ productId }: { productId: string }) {
  const [added, setAdded] = useState(false);

  const handleAdd = () => {
    // fetch API call
    setAdded(true);
  };

  return (
    <button onClick={handleAdd}>
      {added ? '✅ Ditambahkan' : '🛒 Tambah ke Keranjang'}
    </button>
  );
}

💡 Tip: Server Component = chef di dapur (masak di server, kirim hasil matang). Client Component = peralatan di meja makan (interaktif, butuh browser). Gunakan Server Component default, tambah 'use client' hanya saat butuh interaksi.

3. Data Fetching & API Routes

Next.js menyediakan cara berbeda untuk fetching data: server-side (langsung di component), static generation (build time), dan API routes untuk backend logic.

// Server-side fetching (default)
async function UsersPage() {
  const users = await fetch('https://api.example.com/users', {
    cache: 'force-cache' // SSG-like, cache selamanya
    // cache: 'no-store' // SSR, selalu fresh
    // next: { revalidate: 3600 } // ISR, revalidate tiap jam
  }).then(r => r.json());

  return <div>{users.map((u: any) => <div key={u.id}>{u.name}</div>)}</div>;
}

// API Route (app/api/)
// app/api/hello/route.ts
import { NextResponse } from 'next/server';

export async function GET() {
  const data = { message: 'Hello from API!' };
  return NextResponse.json(data);
}

export async function POST(request: Request) {
  const body = await request.json();
  // Process data...
  return NextResponse.json({ success: true });
}

💡 Tip: cache: 'force-cache' = freezer (simpan selamanya). cache: 'no-store' = pasar (selalu fresh). next: { revalidate: 3600 } = kulkas (segar tapi ada tanggal kadaluarsa). Pilih sesuai kebutuhan data-nya.
