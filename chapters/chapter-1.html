Konsep · 8 menit baca

Oke, sebelum kita makin dalam, ada satu hal yang sering bikin pemula overwhelmed dan akhirnya nyerah: ekosistem React yang kayak hutan belantara. Ada Next.js, Gatsby, Remix, React Router, Redux, Zustand, Recoil, TanStack Query... pusing lihatnya semua! 😵‍💫

Tenang. Kita bakal pecahin satu-satu. Bayangin React itu kayak beli mobil. React itu mesinnya. Tapi mesin doang nggak cukup kan? Lo butuh body, ban, jok, AC, sistem audio — itu semua ekosistemnya. Dan yang paling penting: nggak semua harus lo pakai sekaligus.

JSX Deep Dive — Lebih dari Sekadar "HTML di JS" 🎨

Konsep · 12 menit baca

Oke, di Lesson 1 kita udah sempat ngeliat JSX. Tapi sekarang kita bakal bongkar habis-habisan. Soalnya, kalau lo nggak paham JSX sampe akar, lo bakal sering ketemu error aneh yang bikin emosi. 😤

Pertanyaan fundamental: JSX itu apa sih sebenernya? Banyak orang pikir JSX = "HTML yang ditulis di JavaScript". ITUPAYAH BESAR. JSX itu bukan HTML. JSX itu sugar syntax (syntactic sugar) untuk function call React. Yang sebenarnya terjadi di balik layar:

JSX vs React.createElement — Apa yang Sebenarnya Terjadi 
// Yang lo tulis (JSX):
const element = (
  <div className="container">
    <h1>Halo Dunia</h1>
    <p>Ini paragraf</p>
  </div>
);

// Yang sebenarnya di-compile jadi:
const element = React.createElement(
  'div',
  { className: 'container' },
  React.createElement('h1', null, 'Halo Dunia'),
  React.createElement('p', null, 'Ini paragraf')
);

// Hasil akhirnya = JavaScript object:
// {
//   type: 'div',
//   props: {
//     className: 'container',
//     children: [
//       { type: 'h1', props: { children: 'Halo Dunia' } },
//       { type: 'p', props: { children: 'Ini paragraf' } }
//     ]
//   }
// }
//
// JSX itu cuma "cara cantik" nulis React.createElement()
// Biar kita nggak nulis kayak robot 🤖

Aturan-aturan JSX yang WAJIB lo tau (sering bikin error!):

7 Aturan JSX yang Bikin Nggak Error 
// 1. WAJIB return SATU parent element
// BAD: dua element berdampingan tanpa parent
function Bad() {
  return (
    <h1>Judul</h1>
    <p>Paragraf</p>  // ERROR! Adjacent JSX elements
  );
}

// FIX 1: Bungkus dengan div
function Good1() {
  return (
    <div>
      <h1>Judul</h1>
      <p>Paragraf</p>
    </div>
  );
}

// FIX 2: Fragment (lebih baik - nggak nyuntah DOM)
function Good2() {
  return (
    <>
      <h1>Judul</h1>
      <p>Paragraf</p>
    </>
  );
}

// 2. className, BUKAN class
<div className="box">  // YES
<div class="box">      // NO (class adalah keyword JS)

// 3. Self-closing tag WAJIB untuk tag kosong
<img src="logo.png" />   // YES
<input type="text" />    // YES
<img src="logo.png">     // ERROR di JSX

// 4. Expression di curly braces {}, BUKAN string
<p>{count + 1}</p>       // YES - expression
<p>"{count + 1}"</p>     // NO - string literal

// 5. Conditional pakai ternary atau &&
{klik > 10 ? <Bonus /> : null}  // ternary
{showMenu && <Menu />}            // short-circuit

// 6. style butuh object, bukan string
<div style={{ color: 'red', fontSize: '20px' }}>  // YES
<div style="color: red;">                          // ERROR

// 7. List RENDER WAJIB punya key
{items.map(item => (
  <li key={item.id}>{item.name}</li>  // key WAJIB!
))}

JSX Pro Tips yang membedakan pemula sama developer berpengalaman:

JSX Pro Patterns 
// Pattern 1: Spread attributes (copy semua props)
const commonProps = { className: 'btn', disabled: false };
<button {...commonProps} onClick={handleClick}>
  Klik
</button>

// Pattern 2: Conditional className
<div className={`box ${isActive ? 'active' : ''}`}>

// Pattern 3: Inline IIFE untuk complex logic (jarang dipakai tapi tau)
{(() => {
  if (status === 'loading') return <Spinner />;
  if (status === 'error') return <Error />;
  return <Content />;
})()}

// Pattern 4: Render array of elements
const fruits = [
  { id: 1, name: 'Apple' },
  { id: 2, name: 'Banana' },
  { id: 3, name: 'Cherry' }
];

// Map selalu butuh key prop!
// Key harus UNIK dan STABIL (jangan pakai index kalau bisa)
<ul>
  {fruits.map(fruit => (
    <li key={fruit.id}>{fruit.name}</li>
  ))}
</ul>

// Pattern 5: Destructuring di parameter
// Lebih bersih dari: function Card(props) { return props.title }
function Card({ title, desc, image }) {
  return <div><h2>{title}</h2><p>{desc}</p></div>;
}

🧪 RASAIN SENDIRI — conditional rendering + list: Playground di bawah gabungin dua pola paling sering dipakai: {kondisi && ...} buat nampilin/sembunyiin, dan .map() buat render list. Klik tombolnya, terus coba ubah stok jadi 0.

💡 OH TERNYATA GITU! JSX itu nggak ada hubungannya dengan HTML. JSX di-compile jadi React.createElement() — yang nge-return JavaScript object. Browser nggak pernah ngebaca JSX. Yang browser lihat cuma JavaScript murni. JSX cuma bikin hidup kita lebih mudah. 🪄

⚠️ MISTAKE PALING MAHAL! Pakai index sebagai key di list render. Ini kelihatan fine saat development, tapi bakal bikin bug aneh pas lo add/remove/reorder items. Selalu pakai ID yang stabil (database ID, UUID). Kalau nggak ada, generate sendiri!

Composition Patterns — Small to Large 
// STEP 1: Bikin komponen terkecil (atoms)
function Button({ variant, children, ...props }) {
  const styles = {
    primary: 'bg-ink text-white',
    ghost: 'bg-transparent border',
  };
  return (
    <button className={`px-4 py-2 rounded-lg ${styles[variant]}`} {...props}>
      {children}
    </button>
  );
}

function Avatar({ src, alt, size = 40 }) {

// STEP 2: Gabung jadi komponen menengah (molecules)

// Hasil: komponen yang reusable, testable, dan maintainable!

Aturan praktis: kapan pecah komponen?

  • Rule of 3: Kalau lo nulis komponen yang sama (atau mirip) di 3 tempat berbeda, saatnya bikin komponen reusable.
  • Single Responsibility: Satu komponen = satu tugas. Kalau komponen ngerjain terlalu banyak hal, pecah.
  • Maximum depth: Kalau JSX lo punya nested div lebih dari 4-5 level, pertimbangkan pecah jadi sub-komponen.
  • Props too many: Kalau komponen punya >5 props, mungkin terlalu kompleks — pecah jadi 2 komponen.

💡 OH TERNYATA GITU! Composition itu senjata utama React. Bayangin kalau lo bikin Button komponen yang bagus di awal project — lo bisa pakai komponen itu di RATUSAN tempat tanpa duplicate code. Inilah yang bikin React developer produktif: bikin sekali, pakai di mana-mana.

Mini Project — Todo App dari Nol 🚀

Praktik · 20 menit

Cukup teori! Sekarang waktunya bikin sesuatu yang beneran jalan. Kita bakal bikin Todo App — hello world-nya React world. App ini bakal punya: add todo, delete todo, mark complete, filter, dan local storage persistence.

Kenapa Todo App? Karena di app kecil ini, lo udah nyentuh 80% konsep React yang lo butuhin: state, event handling, conditional rendering, list rendering, composition, dan persistence. Ini bukan sembarang latihan — ini fondasi.

Step 1: Basic Todo — Add & List 
import { useState } from 'react';

function TodoApp() {
  const [todos, setTodos] = useState([]);
  const [input, setInput] = useState('');

  const addTodo = () => {
    if (!input.trim()) return; // jangan add yang kosong
    setTodos([...todos, {
      id: Date.now(),
      text: input,
      done: false
    }]);
    setInput(''); // reset input
  };

  const deleteTodo = (id) => {
    setTodos(todos.filter(t => t.id !== id));
  };

  const toggleTodo = (id) => {
    setTodos(todos.map(t =>
      t.id === id ? { ...t, done: !t.done } : t
    ));
  };

  return (
    <div style={{ maxWidth: 500, margin: '0 auto', padding: 20 }}>
      <h1>Todo App 📝</h1>

      {/* Input + Add button */}
      <div style={{ display: 'flex', gap: 8 }}>
        <input
          value={input}
          onChange={(e) => setInput(e.target.value)}
          onKeyDown={(e) => e.key === 'Enter' && addTodo()}
          placeholder="Tambah todo..."
          style={{ flex: 1, padding: 8 }}
        />
        <button onClick={addTodo}>Tambah</button>
      </div>

      {/* Todo list */}
      <ul>
        {todos.map(todo => (
          <li
            key={todo.id}
            style={{
              display: 'flex',
              alignItems: 'center',
              gap: 8,
              padding: 8,
              textDecoration: todo.done ? 'line-through' : 'none',
              opacity: todo.done ? 0.5 : 1
            }}
          >
            <input
              type="checkbox"
              checked={todo.done}
              onChange={() => toggleTodo(todo.id)}
            />
            <span style={{ flex: 1 }}>{todo.text}</span>
            <button onClick={() => deleteTodo(todo.id)}>Hapus</button>
          </li>
        ))}
      </ul>

      {/* Empty state */}
      {todos.length === 0 && (
        <p style={{ color: '#999' }}>Belum ada todo. Yuk tambah!</p>
      )}
    </div>
  );
}

export default TodoApp;

Step 3: Persist ke LocalStorage 
import { useState, useEffect } from 'react';

function TodoApp() {
  // Lazy init: baca dari localStorage saat pertama kali render
  const [todos, setTodos] = useState(() => {
    const saved = localStorage.getItem('todos');
    return saved ? JSON.parse(saved) : [];
  });

  // Save ke localStorage setiap kali todos berubah
  useEffect(() => {
    localStorage.setItem('todos', JSON.stringify(todos));
  }, [todos]);

  // ... sisa kode sama kayak sebelumnya
}

01

Apa Itu React? 🤔

Dasar-dasar · 5 menit baca

Pernah ngalamin moment pas lagi bikin website, terus kamu ubah satu elemen kecil doang — misalnya ganti warna tombol — tapi tiba-tiba seluruh halaman jadi ngeload ulang? Atau pas kode JavaScript-nya udah makin panjang, makin berantakan, dan kamu sendiri bingung mana yang ngubah mana? 😫

Nah, React lahir dari masalah itu. React itu library JavaScript buat bikin user interface yang dibikin oleh Meta (dulu namanya Facebook). Diperkenalin pertama kali tahun 2013, dan sekarang udah dipake jutaan developer di seluruh dunia — termasuk di perusahaan-perusahaan besar kayak Netflix, Airbnb, dan Instagram.

React Component — Bikin Komponen 
// Functional Component (cara modern)
function Welcome({ name }) {
  return <h1>Halo, {name}!</h1>;
}

// Pakai komponennya
<Welcome name="Arif" />
// Output: Halo, Arif!

// Komponen dengan multiple props
function Button({ label, onClick, color = "blue" }) {
  return (
    <button onClick={onClick} style={{ backgroundColor: color }}>
      {label}
    </button>
  );
}

Tapi yang bikin React beda dari yang lain? 🔑 Component-based architecture. Maksudnya apa?

Bayangin kamu lagi bikin LEGO 🧱. Kamu nggak langsung bikin istana raksasa sekaligus kan? Kamu mulai dari bata-bata kecil dulu — satu per satu, lalu disusun jadi bentuk yang lebih besar. Nah, di React, komponen itu kayak bata LEGO-nya. Kamu bikin komponen-komponen kecil yang masing-masing punya tugas sendiri, terus kamu gabung-gabungin jadi halaman yang kompleks.

Setiap komponen ngurus dirinya sendiri — punya data sendiri (state), punya tampilan sendiri, dan nggak nyangkut dengan komponen lainnya. Kodenya jadi rapi, gampang di-test, dan yang paling penting: bisa dipake ulang! 🔄

Oh iya, satu lagi yang penting: React itu declarative. Artinya, kamu cukup ngomongin "mau kayak gimana", bukan "langkah demi langkah harus ngapain". Kamu deskripsikan tampilan UI berdasarkan data/state, terus React yang urus bagaimana cara render-nya ke layar. Keren, kan? 😎

💡 OH TERNYATA GITU! React itu bukan framework ya — dia itu library. Kenapa beda? Karena React fokus ke satu hal aja: rendering UI. Buat routing, state management global, dan hal-hal lainnya, React serahin ke ekosistem (React Router, Zustand, Redux, dll). Jadi React tuh kayak mesin mobil — performa intinya ada di situ, tapi body, AC, dan audio system-nya kamu pilih sendiri 🚗

02

Setup Environment — Gass Keun! 🛠️

Praktik · 10 menit

Oke, sebelum kita mulai ngoding, kita perlu siapin senjatanya dulu. Yang kamu butuhkan cuma satu: Node.js versi 18 ke atas. Node.js itu kayak mesin runtime yang bikin JavaScript bisa jalan di luar browser — dan dia juga bawa npm (Node Package Manager) buat ngelola semua dependensi proyek kita.

Dulu, cara bikin proyek React pake create-react-app. Tapi sayang, tool itu udah deprecated — artinya udah nggak dianjurkan lagi. Sekarang yang jadi bintang baru itu Vite ⚡

Vite itu build tool modern yang super cepat. Kenapa? Karena dia pake ES modules secara native, jadi proses build dan hot reload tuh hampir instan. Bener-bener beda 180 derajat sama Create React App yang dulu sering lama banget pas pertama kali jalan. 💨

Setelah proyeknya jadi, kamu tinggal jalanin development server-nya. Vite bakal auto-open browser dan nampilin aplikasi React default. Terus setiap kali kamu ubah kode — BAM! — perubahannya langsung keupdate di browser tanpa perlu refresh manual. Fitur ini namanya Hot Module Replacement (HMR), dan trust me, ini bikin pengalaman development terasa jauh lebih smooth 🧈

Terminal — Bikin Proyek React Baru dengan Vite 
# Install Node.js dulu kalau belum ada ya!
# Terus bikin proyek React baru pake Vite
npm create vite@latest my-react-app -- --template react

# Masuk ke folder proyek
cd my-react-app

# Install semua dependensinya
npm install

# Jalankan development server
npm run dev

# Buka browser di http://localhost:5173

Virtual DOM — Rahasia React yang Bikin Cepat ⚡

Konsep inti · 8 menit baca

Okay, ini bagian yang sering bikin orang "oh ternyata gitu!" 🤯 Jadi dulu sebelum React, pas kita mau update halaman web, kita langsung utik-utik DOM-nya. Masalahnya? DOM itu MAHAL.

Setiap kali kamu ngubah satu elemen doang di DOM, browser harus nge-reflow dan nge-repaint seluruh halaman. Bayangin kamu punya form dengan 50 input, terus tiap ketik satu huruf, browser harus re-calculate layout SEMUA elemen. Hasilnya? Aplikasi jadi lag dan lemot. 🐌

Virtual DOM — Cara React Update 
// React bikin Virtual DOM = copyan DOM di memory
// Cuma update bagian yang berubah, bukan reload semua

function App() {
  const [count, setCount] = useState(0);
  return (
    <div>
      <h1>Counter: {count}</h1>
      <p>Static text</p>
      <button onClick={() => setCount(count + 1)}>+</button>
    </div>
  );
}

// Setelah setCount(1):
// React compare Virtual DOM lama vs baru
// Cuma <h1> yang di-update di real DOM
// <p> dan <button> tidak disentuh = efisien!

Nah, di sinilah Virtual DOM jadi pahlawan! 💪

Gini cara kerjanya: React bikin salinan DOM di memori — ini yang disebut Virtual DOM. Salinan ini super ringan karena cuma objek JavaScript biasa, bukan DOM asli yang berat. Setiap kali state berubah, React bikin Virtual DOM baru, lalu bandingin sama versi sebelumnya. Proses nyari perbedaan ini namanya diffing.

Setelah tau bedanya di mana, React baru update bagian DOM yang bener-bener berubah doang — nggak semuanya. Proses ini disebut reconciliation. Hasilnya? Update yang super efisien! 🎯

TL;DR dari prosesnya:

  1. State berubah 🔄
  2. React bikin Virtual DOM baru 📋
  3. React bandingin Virtual DOM lama vs baru, cari bedanya 🔍
  4. Hanya DOM yang beda yang di-update di layar ✨

Satu konsep penting lagi yang perlu kamu tahu: one-way data flow. Di React, data itu mengalir dari atas ke bawah — dari komponen parent ke komponen child lewat sesuatu yang namanya props.

🧳 Analogi Koper Travel: Bayangin kamu punya satu koper bagus (itu component-nya). Tiap anggota keluarga bisa isi koper yang sama dengan barang-barang berbeda — baju kamu, mainan adik, alat masak ibu. Tapi aturan utamanya: kamu cuma bisa masukin barang ke koper dari luar. Nggak bisa minta koper buka sendiri dan ngeliat isinya. Isi koper itu read-only — kalau mau ganti isi, ya harus buka koper lagi dari luar. Itulah props!

Props bikin alur datanya gampang ditebak dan gampang di-debug, dibanding framework yang pake two-way binding yang kadang bikin bingung "ini datanya berubah dari mana sih?!" 😅

Perbandingan: Cara Lama vs Cara React 
// ❌ Cara lama — langsung utik DOM manual
// Setiap perubahan trigger repaint
document.getElementById('title').textContent = 'Halo Baru';
document.getElementById('counter').textContent = '1';
// Repaint terjadi 2 kali — boros! 😤

// ✅ Cara React — Virtual DOM ngurus semuanya
// React hitung diff, lalu update DOM sekali aja
function App() {
  const [title, setTitle] = useState('Halo');
  const [count, setCount] = useState(0);

  // State berubah → Virtual DOM di-update →
  // React hitung diff → DOM aktual di-update sekali ✅
  return (
    <div>
      <h1>{title}</h1>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        Tambah
      </button>
    </div>
  );
}

💡 ANALOGI GAMBLANG: Virtual DOM itu kayak draft di aplikasi edit foto. Coba bayangin: kamu mau edit foto, tapi alih-alih edit langsung di foto asli, kamu buat salinan virtual-nya dulu. Di salinan virtual itu kamu bisa edit sana-sini sesuka hati — crop, ganti warna, tambah teks — tanpa ngaruh ke foto asli. Setelah kamu puas, barulah hasil akhirnya di-flatten ke foto asli. Satu kali apply, semua berubah. Itu yang dilakuin React sama DOM! 📸

Hello World Pertamamu! 🎉

Praktik · 7 menit

Sekarang waktunya seru-seruannya! Kita bakal nulis kode React pertamamu. Beneran, langsung coba! 💻

Buka file src/App.jsx di proyek Vite yang tadi kamu bikin, terus ganti semua isinya dengan kode di bawah ini.

Tapi sebelum itu, ada baiknya kamu paham dulu mindset-nya. Komponen React itu pada dasarnya cuma fungsi JavaScript biasa yang nge-return elemen UI. Yap, sesimpel itu! Kamu udah bisa JavaScript? Kamu udah selangkah lebih deket sama React 🎯

Di contoh ini, kita bakal pake useState — ini adalah React Hook yang bikin komponen fungsional kita punya state (memori). Hook ini nge-return dua hal: nilai state-nya sekarang, dan fungsi buat ngubahnya. Ini fondasi dari semua interaktivitas di React!

src/App.jsx — Hello World + Interaksi 
import { useState } from 'react';
import './App.css';

function App() {
  // State buat nyimpan nama — defaultnya "Dunia"
  const [nama, setNama] = useState('Dunia');
  // State buat ngitung berapa kali user klik
  const [klik, setKlik] = useState(0);

  const handleClick = () => {
    setKlik(klik + 1);
  };

  const handleNameChange = (e) => {
    setNama(e.target.value);
  };

  return (
    <div style={{ padding: '2rem', textAlign: 'center' }}>
      <h1 style={{ color: '#0B0B0B', fontSize: '2.5rem' }}>
        Halo, {nama}! 🎉
      </h1>
      <p style={{ color: '#9ca3af', marginTop: '1rem' }}>
        Ini aplikasi React pertamamu. Keren, kan?
      </p>

      <div style={{ marginTop: '2rem' }}>
        <input
          type="text"
          value={nama}
          onChange={handleNameChange}
          placeholder="Ketik namamu di sini..."
          style={{
            padding: '0.75rem 1rem',
            borderRadius: '0.75rem',
            border: '2px solid #D1D1D1',
            background: '#F5F4F1',
            color: '#0B0B0B',
            fontSize: '1rem',
            width: '250px'
          }}
        />
      </div>

      <div style={{ marginTop: '1.5rem' }}>
        <button
          onClick={handleClick}
          style={{
            padding: '0.75rem 1.5rem',
            borderRadius: '0.75rem',
            background: '#0B0B0B',
            color: '#FFFFFF',
            fontWeight: 'bold',
            border: 'none',
            cursor: 'pointer',
            fontSize: '1rem'
          }}
        >
          Klik aku! ({klik} kali)
        </button>
      </div>

      {klik >= 10 && (
        <p style={{ color: '#f59e0b', marginTop: '1rem' }}>
          🎯 Wah kamu udah klik {klik} kali! Semangat belajar React!
        </p>
      )}
    </div>
  );
}

export default App;

🧪 GILIRAN LO — coba langsung di browser! Playground di bawah ini React beneran. Edit kodenya, klik ▶ Jalankan, lihat hasilnya berubah. Perhatiin: nama dilempar sebagai prop, dan {umur + 1} itu expression JS yang dieksekusi pas render.

GIMANA? KEREN NGGAK?! 🤩 Coba kamu perhatiin beberapa hal keren dari kode di atas:

1. Reaktivitas Otomatis. Pas kamu ketik nama di input, teks "Halo, {nama}!" langsung berubah. Nggak perlu manual update DOM — React ngurus semuanya. Itu kekuatan state!

2. Conditional Rendering. Lihat bagian {klik >= 10 && (...)}? Itu artinya pesan kuning cuma muncul kalau kamu udah klik 10 kali atau lebih. JavaScript biasa yang langsung jadi logika UI!

3. Fungsi sebagai Komponen. Lihat? function App() — cuma fungsi biasa yang nge-return JSX. Simpel banget!

🧪 COBA UTIK-UTIK! Sekarang coba beberapa eksperimen ini biar makin ngerasain React:

  • Ganti default name dari 'Dunia' jadi nama kamu sendiri
  • Tambahin state baru buat nge-tracking warna favorit
  • Ubah threshold dari 10 klik jadi 50 klik — lihat apa yang berubah

Nggak perlu takut error — itu bagian dari belajar! 🚀

⚠️ KESALAHAN YANG SERING TERJADI: Lupa import useState dari 'react'? Wajar banget, hampir semua developer pernah ngalamin ini 😂 Error yang muncul biasanya: "useState is not defined". Selalu cek baris import di atas file ya!

Level Senior: Mental Model React — UI = f(state) 🧠

Level Senior · 10 menit baca

Lo udah bisa nulis komponen. Tapi ada satu pergeseran cara mikir yang misahin orang yang "bisa React" sama orang yang "mikir React". Kalau lo dateng dari jQuery atau JavaScript vanilla, otak lo terbiasa imperatif: "ambil elemen ini, ubah teksnya, tambahin class, sembunyiin yang itu." Langkah demi langkah, lo yang atur DOM-nya sendiri.

React itu deklaratif. Lo nggak mikir "ubah DOM". Lo mikir: "untuk state seperti ini, UI-nya harus kelihatan seperti apa?" Lalu React yang urus caranya nyampe ke sana. Persamaan mental yang wajib lo tanam: UI = f(state) — tampilan itu hasil (fungsi) dari state. State berubah, React panggil ulang fungsi komponennya, dapet deskripsi UI baru, terus dia sendiri yang ngitung perubahan DOM minimal. Lo cukup jujur mendeskripsikan "kondisi akhir", bukan "urutan langkah".

🍜 Analogi warteg vs chef pribadi: Cara imperatif itu kayak lo masuk dapur dan mandu chef langkah demi langkah — "ambil wajan, nyalain kompor, tuang minyak, masukin bawang..." Capek dan gampang salah urutan. Cara deklaratif itu kayak pesan di warteg: lo bilang "mau nasi + ayam geprek + es teh" (itu state akhirnya), dan dapur yang mikirin gimana bikinnya. Lo cuma deskripsikan hasil yang lo mau, bukan resepnya.

Imperatif (lo yang atur DOM) vs Deklaratif (React yang atur) 
// ❌ IMPERATIF — lo yang perintah DOM langkah demi langkah
const btn = document.getElementById('btn');
let suka = false;
btn.addEventListener('click', () => {
  suka = !suka;                          // 1. ubah data
  btn.textContent = suka ? '❤️ Disuka' : '🤍 Suka';  // 2. update teks manual
  btn.classList.toggle('active', suka);  // 3. update class manual
  // Tiap penambahan fitur = tambah langkah manual. Gampang lupa sync. 😵
});

// ✅ DEKLARATIF — lo deskripsikan "UI untuk state ini", React yang sync
function TombolSuka() {
  const [suka, setSuka] = React.useState(false);
  // Nggak ada 'ubah DOM'. Cuma: "kalau suka true, tampilannya begini".
  return (
    <button className={suka ? 'active' : ''} onClick={() => setSuka(!suka)}>
      {suka ? '❤️ Disuka' : '🤍 Suka'}
    </button>
  );
}
// Mau tambah counter like? Tambah 1 baris di JSX. React yang jaga konsistensi.

Kenapa ini scale? Di app kecil, bedanya keliatan sepele. Tapi bayangin dashboard dengan 40 elemen yang saling gantung ke satu state. Cara imperatif: tiap state berubah, lo harus inget semua tempat yang perlu di-sync manual — dan satu yang kelupaan = bug "UI-nya nggak update". Cara deklaratif: lo cuma punya satu sumber kebenaran (state), dan React jamin seluruh UI selalu cerminan dari state itu. Nggak ada "UI dan data beda cerita". Inilah kenapa senior obsesif soal "di mana state ini seharusnya hidup" — karena begitu state-nya bener, UI-nya ngikut otomatis.

Cara senior baca sebuah render: Pas debugging, senior nggak tanya "elemen mana yang salah di-update?" Dia tanya "state apa yang salah, dan komponen apa yang seharusnya render ulang saat state itu berubah?" Bug UI di React hampir selalu = bug state, bukan bug DOM. Geser fokus lo dari "manipulasi tampilan" ke "manajemen state", dan lo udah mikir kayak senior.

💡 OH TERNYATA GITU! Kalimat sakti yang dipegang developer React berpengalaman: "Jangan sentuh DOM, ubah state." Setiap kali tangan lo gatel mau document.getElementById(...) di dalam komponen React buat ngubah tampilan — berhenti. Tanya: "state apa yang ngedeskripsikan kondisi ini?" 99% kasus, jawabannya useState, bukan manipulasi DOM langsung. 🎯

⚠️ JANGAN TERJERAT! Godaan terbesar pemula eks-jQuery: pas UI nggak update, refleks nyari document.querySelector buat "maksa" ubah tampilan. Itu ngelawan React — lo dan React bakal berebut kontrol DOM, dan hasilnya bug yang lebih parah. Sumber kebenaran cuma SATU: state. UI ngikut, bukan sebaliknya.

⚠️ Kesalahan Umum — 6 Jebakan Pemula React

Debugging · 10 menit baca

Enam error ini bakal lo temuin di minggu pertama belajar React — dijamin. Kabar baiknya: begitu lo hafal polanya, lo bisa fix dalam 5 detik sambil ketawa. Format tiap kesalahan: kode salah → kenapa salah → kode benar.

Kesalahan #1 — Return dua elemen tanpa satu pembungkus

#1 Adjacent JSX elements 
// ❌ SALAH — dua elemen bersebelahan, nggak ada parent
function App() {
  return (
    <h1>Judul</h1>
    <p>Isi</p>   // Error: "Adjacent JSX elements must be wrapped..."
  );
}
// KENAPA: return cuma boleh nge-return SATU nilai. JSX di atas =
// dua object element sekaligus → JS bingung, mana yang di-return?

// ✅ BENAR — bungkus pakai Fragment (nggak nambah div di DOM)
function App() {
  return (
    <>
      <h1>Judul</h1>
      <p>Isi</p>
    </>
  );
}

Kesalahan #2 — class padahal harusnya className

#2 class vs className 
// ❌ SALAH — copas dari HTML, pakai `class`
<div class="kartu">Halo</div>   // warning + styling nggak nempel

// KENAPA: JSX itu JavaScript, dan `class` adalah keyword JS (buat bikin
// class/OOP). Jadi React pakai `className` biar nggak bentrok. Sama juga:
// `for` di label jadi `htmlFor`.

// ✅ BENAR
<div className="kartu">Halo</div>
<label htmlFor="email">Email</label>

Kesalahan #3 — Bingung {} untuk expression vs "" untuk string

#3 Curly braces vs quotes 
const nama = 'Arif';
const umur = 20;

// ❌ SALAH — teksnya kebaca literal, bukan nilai variabel
<p>Halo nama</p>            // tampil: "Halo nama"
<img src={logo.png} />      // Error: logo is not defined (dianggap variabel)
<input value="umur" />      // value literal string "umur"

// KENAPA: teks/atribut TANPA {} = string literal apa adanya. Pakai {} kalau
// mau menyisipkan NILAI JavaScript (variabel, hitungan, panggilan fungsi).

// ✅ BENAR
<p>Halo {nama}</p>          // tampil: "Halo Arif"
<img src="logo.png" />      // src memang string → pakai quotes
<p>Umur tahun depan: {umur + 1}</p>   // expression di dalam {}

Kesalahan #4 — Manggil komponen kayak fungsi biasa / nama huruf kecil

#4 Foo() vs <Foo /> 
function Profil() { return <h2>Arif</h2>; }

// ❌ SALAH — dipanggil kayak fungsi biasa
function App() {
  return <div>{Profil()}</div>;   // "jalan" tapi bukan komponen sungguhan:
}                                   // nggak punya state/lifecycle sendiri

// ❌ SALAH — nama huruf kecil
function profil() { return <h2>Arif</h2>; }
<profil />   // React kira ini tag HTML <profil>, bukan komponenmu!

// KENAPA: React bedain komponen dari tag HTML lewat huruf BESAR di awal.
// Dan render komponen itu <Profil />, bukan Profil() — biar React yang
// kelola sebagai unit (punya state, key, re-render sendiri).

// ✅ BENAR
function Profil() { return <h2>Arif</h2>; }
function App() { return <div><Profil /></div>; }

Kesalahan #5 — Lupa import

#5 Forgotten import 
// ❌ SALAH — pakai useState tanpa import
function App() {
  const [n, setN] = useState(0);   // Error: useState is not defined
  return <button onClick={() => setN(n + 1)}>{n}</button>;
}

// KENAPA: hook & komponen dari file lain harus diimpor dulu. Error paling
// sering pemula: "X is not defined" = 90% lupa baris import di atas.

// ✅ BENAR
import { useState } from 'react';       // hook bawaan React
import Tombol from './Tombol';          // komponen sendiri (default export)
import { Badge } from './ui';           // named export

Kesalahan #6 — Nge-render object langsung sebagai teks

#6 Objects are not valid as a React child 
const user = { nama: 'Arif', role: 'Dev' };

// ❌ SALAH — masukin object utuh ke JSX
<p>{user}</p>   // Error: "Objects are not valid as a React child"

// KENAPA: JSX bisa nampilin string, number, atau element — TAPI bukan object
// mentah. React nggak tau harus nampilin object jadi apa.

// ✅ BENAR — ambil field-nya, atau map jadi element
<p>{user.nama} — {user.role}</p>
// Kalau array of object: pakai .map jadi <li> (jangan render array object polos)

⚠️ JANGAN TERJERAT! Baca pesan error-nya! React 18 kasih error yang jelas banget — "Adjacent JSX elements...", "Objects are not valid as a React child", "X is not defined". 6 dari 6 kesalahan di atas ketebak dari kalimat pertama error-nya. Jangan panik, jangan langsung Google — baca dulu. 🔍

Kerjain di kepala dulu, baru buka solusinya. Jujur ya. 😄

✏️ Latihan 1: Benerin JSX yang error klik untuk lihat solusi

Kode ini punya 3 error. Temukan dan perbaiki tanpa lihat solusi dulu:

Soal — cari 3 error
function Kartu() {
  return (
    <h2 class="judul">Halo</h2>
    <p>Nama saya nama</p>
  );
}

Solusi: (1) dua elemen bersebelahan tanpa pembungkus, (2) class harusnya className, (3) nama harusnya {nama} (dan variabelnya perlu didefinisikan).

Versi benar
function Kartu({ nama }) {
  return (
    <>
      <h2 className="judul">Halo</h2>
      <p>Nama saya {nama}</p>
    </>
  );
}
✏️ Latihan 2: Komponen Salam pakai prop klik untuk lihat solusi

Bikin komponen Salam yang nerima prop nama dan waktu (misal "pagi"), lalu render "Selamat pagi, Arif!". Render tiga kali dengan nama berbeda.

Solusi
function Salam({ nama, waktu }) {
  return <p>Selamat {waktu}, {nama}! 👋</p>;
}

function App() {
  return (
    <div>
      <Salam nama="Arif" waktu="pagi" />
      <Salam nama="Sinta" waktu="siang" />
      <Salam nama="Budi" waktu="malam" />
    </div>
  );
}

💡 Perhatiin: satu komponen, dipakai 3x dengan prop beda = inti "reusable component".

✏️ Latihan 3: Conditional rendering status stok klik untuk lihat solusi

Bikin komponen Produk yang nerima prop nama dan stok (number). Kalau stok > 0 tampilkan "Tersedia (N)", kalau 0 tampilkan "Habis". Pakai ternary.

Solusi
function Produk({ nama, stok }) {
  return (
    <div>
      <strong>{nama}</strong>{' '}
      <span style={{ color: stok > 0 ? 'green' : 'red' }}>
        {stok > 0 ? `Tersedia (${stok})` : 'Habis'}
      </span>
    </div>
  );
}

💡 {kondisi ? A : B} = ternary, cara paling umum conditional rendering di JSX.

🎯 CHALLENGE — KOMPONEN PROFIL DARI PROPS

Bikin komponen <Profil /> yang nampilin nama dan role dari props, lalu render dua kartu di dalam App: Arif — Frontend Dev dan Sinta — Backend Dev. Starter di bawah masih belum lengkap (cuma 1 kartu, role-nya hilang). Perbaiki sampai semua test lulus. Klik ▶ Test.

function Profil({ nama, role }) {
