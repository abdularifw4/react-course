01

Context API Deep Dive

Fundamental · 8 menit baca

Context API adalah solusi bawaan React untuk state management. Context terdiri dari createContext(), Provider, dan useContext(). Context cocok untuk state global sederhana seperti tema, bahasa, autentikasi, dan preferensi user.

Namun Context punya kelemahan: setiap perubahan value akan me-rerender semua konsumen, bahkan yang tidak menggunakan bagian data yang berubah. Ini bisa menjadi performance issue di aplikasi besar dengan state kompleks.

Untuk mengatasi ini, gunakan useReducer bersama Context agar update state lebih terprediksi, atau split Context menjadi beberapa Context kecil yang masing-masing menangani concern berbeda (misalnya AuthContext terpisah dari ThemeContext).

Context API + useReducer
import { createContext, useContext, useReducer } from 'react';

// Definisikan state dan actions
const CartContext = createContext();

const initialState = { items: [], total: 0 };

function cartReducer(state, action) {
  switch (action.type) {
    case 'ADD_ITEM':
      return {
        ...state,
        items: [...state.items, action.payload],
        total: state.total + action.payload.price,
      };
    case 'REMOVE_ITEM':
      return {
        ...state,
        items: state.items.filter(i => i.id !== action.payload.id),
        total: state.total - action.payload.price,
      };
    case 'CLEAR':
      return initialState;
    default:
      return state;
  }
}

function CartProvider({ children }) {
  const [state, dispatch] = useReducer(cartReducer, initialState);
  return (
    <CartContext.Provider value={{ state, dispatch }}>
      {children}
    </CartContext.Provider>
  );
}

function useCart() {
  const ctx = useContext(CartContext);
  if (!ctx) throw new Error('useCart harus di dalam CartProvider');
  return ctx;
}

// Penggunaan di komponen manapun:
function AddToCartButton({ product }) {
  const { dispatch } = useCart();
  return (
    <button onClick={() => dispatch({ type: 'ADD_ITEM', payload: product })}>
      Tambah ke Keranjang
    </button>
  );
}

💡 Analogi: Context API seperti papan pengumuman di kantor. Semua orang bisa lihat info di papan (state global), tapi kalau pengumuman sering berubah, semua orang harus membaca ulang (rerender). Untuk data yang sering berubah, perlu solusi yang lebih efisien.

02

Zustand

Library Populer · 8 menit baca

Zustand adalah state management library yang sangat ringan (~1KB), simple, dan powerful. Tidak perlu Provider wrapper, tidak ada boilerplate, dan mendukung selector untuk optimasi rerender.

Zustand menggunakan store pattern di mana kamu mendefinisikan state dan actions dalam satu objek. Komponen bisa subscribe ke seluruh store atau hanya bagian tertentu menggunakan selector, menghindari rerender yang tidak perlu.

Keunggulan Zustand: API minimalis, mendukung devtools, middleware (persist, immer, devtools), bisa digunakan di luar komponen (di utility functions), dan support TypeScript dengan baik.

Zustand Store
import { create } from 'zustand';

// Buat store — tidak perlu Provider!
const useBearStore = create((set, get) => ({
  bears: 0,
  increase: () => set((state) => ({ bears: state.bears + 1 })),
  decrease: () => set((state) => ({ bears: state.bears - 1 })),
  reset: () => set({ bears: 0 }),
  setBears: (count) => set({ bears: count }),
}));

// Penggunaan di komponen
function BearCounter() {
  // Selector — hanya rerender jika bears berubah
  const bears = useBearStore((state) => state.bears);
  const increase = useBearStore((state) => state.increase);

  return <div>
    <h1>{bears} beruang</h1>
    <button onClick={increase}>Tambah Beruang</button>
  </div>;
}

// Store dengan persist middleware (simpan ke localStorage)
const useUserStore = create(
  persist(
    (set) => ({
      name: '',
      theme: 'dark',
      setName: (name) => set({ name }),
      setTheme: (theme) => set({ theme }),
    }),
    { name: 'user-settings' }
  )
);

💡 Analogi: Zustand seperti lemari es bersama di kos-kosan. Semua penghuni bisa ambil dan simpan makanan (state) kapan saja tanpa perlu izin (Provider). Masing-masing penghuni bisa langsung akses tanpa melalui perantara.

03

Redux Toolkit & Jotai

Library Lanjutan · 9 menit baca

Redux Toolkit (RTK) adalah cara modern menggunakan Redux. RTK menyederhanakan setup Redux dengan createSlice (otomatis buat actions dan reducers), createAsyncThunk (handle async), dan createStore (simplified store setup).

Jotai mengambil pendekatan berbeda — atomic state management. Alih-alih satu store besar, state dipecah menjadi atom-atom kecil yang independen. Komponen subscribe ke atom tertentu, sehingga rerender sangat optimal.

RTK cocok untuk aplikasi besar dengan tim banyak developer karena struktur yang ketat dan devtools yang lengkap. Jotai cocok untuk aplikasi yang mengutamakan fleksibilitas dan performa granular.

Redux Toolkit & Jotai
// Redux Toolkit — createSlice
import { createSlice, configureStore } from '@reduxjs/toolkit';

const counterSlice = createSlice({
  name: 'counter',
  initialState: { value: 0 },
  reducers: {
    increment: (state) => { state.value += 1; },
    decrement: (state) => { state.value -= 1; },
    incrementBy: (state, action) => { state.value += action.payload; },
  },
});

const store = configureStore({ reducer: { counter: counterSlice.reducer } });
export const { increment, decrement, incrementBy } = counterSlice.actions;

// Jotai — Atomic state management
import { atom, useAtom } from 'jotai';

// Definisikan atom (unit state terkecil)
const countAtom = atom(0);
const doubledCountAtom = atom((get) => get(countAtom) * 2);

// Penggunaan
function Counter() {
  const [count, setCount] = useAtom(countAtom);
  const doubled = useAtomValue(doubledCountAtom);
  return (
    <div>
      <span>Count: {count} (Doubled: {doubled})</span>
      <button onClick={() => setCount(c => c + 1)}>+1</button>
    </div>
  );
}

💡 Analogi: Redux Toolkit seperti sistem filing kantor yang terorganisir rapi (struktur ketat, dokumentasi lengkap). Jotai seperti sticky notes yang bisa ditempel di mana saja — fleksibel, ringan, dan bisa diakses langsung tanpa melalui sistem filing.

04

Kapan Pakai Apa?

Decision Guide · 6 menit baca

Memilih state management yang tepat sangat penting. useState cukup untuk state lokal komponen. Context API cocok untuk state global sederhana (tema, auth, i18n) di aplikasi kecil-menengah.

Zustand adalah pilihan terbaik untuk kebanyakan aplikasi modern: API minimalis, tanpa Provider, performa baik, dan learning curve rendah. Gunakan untuk state global yang kompleks atau sering berubah.

Redux Toolkit untuk aplikasi enterprise dengan tim besar yang butuh struktur ketat dan tooling lengkap. Jotai untuk state yang sangat granular dan performa-critical. Keduanya bisa dicampur: Zustand untuk global, useState untuk lokal, Context untuk theme.

Decision Guide
# Decision tree state management React:

# 1. State hanya dipakai 1 komponen?
#    ✓ => useState / useReducer
#
# 2. State dipakai beberapa komponen dekat (parent-child)?
#    ✓ => Lift state up + pass via props
#
# 3. State global tapi sederhana (theme, auth, lang)?
#    ✓ => Context API
#
# 4. State global kompleks, sering berubah, perlu selector?
#    ✓ => Zustand (rekomendasi default)
#
# 5. Aplikasi enterprise, tim besar, butuh time travel debugging?
#    ✓ => Redux Toolkit
#
# 6. State sangat granular, perlu optimasi rerender ekstrem?
#    ✓ => Jotai

⚠️ Perhatian: Jangan over-engineer! Mulai dengan useState, lalu naik ke Context atau Zustand hanya ketika benar-benar dibutuhkan. Menambah state management library memiliki cost: bundle size, complexity, dan learning curve.
