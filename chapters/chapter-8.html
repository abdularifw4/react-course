Chapter 8

State Management

1. Context API Deep Dive

Context API sudah ada sejak React 16.3 dan cukup untuk aplikasi menengah. Cocok untuk theme, auth, locale, dan global settings. Tapi untuk state yang sering berubah, library dedicated lebih efisien.

import { createContext, useContext, useReducer, useMemo } from 'react';

// Typed Context
interface Todo {
  id: string; text: string; done: boolean;
}
interface TodoContextType {
  todos: Todo[];
  addTodo: (text: string) => void;
  toggleTodo: (id: string) => void;
}

const TodoContext = createContext<TodoContextType | null>(null);

function TodoProvider({ children }: { children: React.ReactNode }) {
  const [todos, dispatch] = useReducer(todoReducer, []);

  const value = useMemo(() => ({
    todos,
    addTodo: (text: string) => dispatch({ type: 'ADD', payload: text }),
    toggleTodo: (id: string) => dispatch({ type: 'TOGGLE', payload: id }),
  }), [todos]);

  return <TodoContext.Provider value={value}>{children}</TodoContext.Provider>;
}

function useTodos() {
  const ctx = useContext(TodoContext);
  if (!ctx) throw new Error('useTodos harus dalam TodoProvider');
  return ctx;
}

💡 Tip: Context + useReducer + custom hook = setup state management yang solid tanpa library. Tapi ingat: setiap context value berubah = semua konsumer re-render. Untuk data besar/sering berubah, pertimbangkan Zustand atau Jotai.

2. Zustand — Simple & Powerful

Zustand adalah state management library yang minimalis. Tidak perlu Provider, tidak perlu boilerplate. Sangat populer di komunitas React karena kesederhanaannya.

import { create } from 'zustand';

// Buat store — tanpa Provider!
interface CartStore {
  items: CartItem[];
  addItem: (item: CartItem) => void;
  removeItem: (id: string) => void;
  total: () => number;
}

const useCartStore = create<CartStore>((set, get) => ({
  items: [],
  addItem: (item) => set((state) => ({
    items: [...state.items, item]
  })),
  removeItem: (id) => set((state) => ({
    items: state.items.filter(i => i.id !== id)
  })),
  total: () => get().items.reduce((sum, i) => sum + i.price * i.qty, 0),
}));

// Pakai di komponen manapun — tanpa Provider!
function CartButton() {
  const items = useCartStore(s => s.items);
  const addItem = useCartStore(s => s.addItem);
  return (
    <button onClick={() => addItem({ id: '1', name: 'Produk', price: 50000, qty: 1 })}>
      Tambah ({items.length} item)
    </button>
  );
}

💡 Tip: Zustand itu seperti brankas mini — bisa diakses dari mana saja tanpa perlu bungkus Provider. Selector <code>s => s.items</code> memastikan komponen hanya re-render saat items berubah, bukan saat property lain berubah.

3. Kapan Pakai Apa? — Decision Guide

Pemilihan state management yang tepat tergantung skala dan kebutuhan aplikasi.

// Decision tree:
//
// 1. State lokal 1 komponen?
//    → useState
//
// 2. State dibagi 2-3 komponen dekat?
//    → Lift state up + props
//
// 3. State global tapi jarang berubah (theme, auth, locale)?
//    → Context API
//
// 4. State global, sering berubah, kompleks?
//    → Zustand (recommended) atau Jotai
//
// 5. Aplikasi enterprise, banyak developer, strict patterns?
//    → Redux Toolkit

// Contoh: Mix semua di satu app
function App() {
  return (
    <ThemeProvider>         {/* Context untuk theme */}
      <AuthProvider>        {/* Context untuk auth */}
        <Router>
          <CartPage />       {/* Zustand untuk cart */}
          <UserPage />        {/* Zustand untuk user profile */}
        </Router>
      </AuthProvider>
    </ThemeProvider>
  );
}

💡 Tip: Jangan over-engineer! Mulai dari useState, naik ke Context kalau perlu share, baru ke Zustand kalau Context terlalu sering re-render. Redux Toolkit hanya kalau tim butuh strict conventions.
