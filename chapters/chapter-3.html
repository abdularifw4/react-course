React Hooks Dasar

1. useState — Mental Model State di React

useState adalah hook paling fundamental di React. Hook ini memungkinkan komponen fungsional untuk memiliki state internal — data yang bisa berubah dan memicu re-render.

Sebelum hooks diperkenalkan di React 16.8, hanya komponen class yang bisa memiliki state. Dengan useState, komponen fungsional kini sama powerful-nya dengan komponen class.

Gunakan useState ketika kamu punya data yang berubah karena interaksi user — seperti input teks, toggle, counter, atau selected item.

// Sintaks dasar useState
const [namaState, setNamaState] = useState(nilaiAwal);

// Contoh: Counter
import { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Tambah</button>
      <button onClick={() => setCount(count - 1)}>Kurang</button>
      <button onClick={() => setCount(0)}>Reset</button>
    </div>
  );
}

// Contoh: Multiple states
function Form() {
  const [nama, setNama] = useState('');
  const [email, setEmail] = useState('');
  const [submitted, setSubmitted] = useState(false);

  const handleSubmit = (e) => {
    e.preventDefault();
    setSubmitted(true);
  };

  if (submitted) {
    return <p>Terima kasih, {nama}!</p>;
  }

  return (
    <form onSubmit={handleSubmit}>
      <input value={nama} onChange={e => setNama(e.target.value)} />
      <input value={email} onChange={e => setEmail(e.target.value)} />
      <button type="submit">Kirim</button>
    </form>
  );
}

💡 Tip: Bayangkan useState sebagai kotak penyimpanan kecil. setCount = mengganti isi kotak. Setiap kali isi berubah, React otomatis "menggambar ulang" komponen. Satu komponen bisa punya banyak useState — seperti banyak kotak penyimpanan.

2. useEffect — Menggantikan Lifecycle Methods

useEffect adalah hook yang menjalankan side effects — operasi yang terjadi di luar proses rendering normal. Contohnya: fetching data, subscription, DOM manipulation, dan timer.

useEffect menggantikan tiga lifecycle methods komponen class: componentDidMount, componentDidUpdate, dan componentWillUnmount.

import { useState, useEffect } from 'react';

// 1. Dependency array kosong — jalan sekali saat mount
function UserProfile({ userId }) {
  const [user, setUser] = useState(null);

  useEffect(() => {
    fetch(`/api/users/${userId}`)
      .then(res => res.json())
      .then(data => setUser(data));
  }, [userId]); // Hanya jalan saat userId berubah
}

// 2. Cleanup function — cegah memory leak
function Timer() {
  const [seconds, setSeconds] = useState(0);

  useEffect(() => {
    const timer = setInterval(() => {
      setSeconds(s => s + 1);
    }, 1000);

    // Cleanup: jalan saat component unmount
    return () => clearInterval(timer);
  }, []); // Kosong = mount sekali, cleanup saat unmount
}

// 3. Multiple effects
function Dashboard() {
  const [data, setData] = useState([]);
  const [theme, setTheme] = useState('dark');

  // Effect untuk data
  useEffect(() => {
    fetch('/api/data').then(r => r.json()).then(setData);
  }, []);

  // Effect untuk theme (side effect terpisah)
  useEffect(() => {
    document.body.className = theme;
  }, [theme]);
}

💡 Tip: Dependency array adalah KUNCI memahami useEffect. [] = sekali saja (mount). [dep] = saat dep berubah. Tanpa array = setiap render (berbahaya!). Bayangkan seperti alarm: tanpa array = alarm setiap detik, [] = alarm sekali, [waktu] = alarm saat waktu berubah.

3. useRef — Referensi ke DOM dan Nilai Mutable

useRef memberikan kamu sebuah referensi persisten yang bertahan sepanjang lifecycle komponen. Dua kegunaan utama: mengakses elemen DOM langsung, dan menyimpan nilai yang tidak memicu re-render.

Berbeda dengan state, mengubah ref.current tidak akan memicu re-render. Ini membuatnya ideal untuk timer ID, previous value, atau DOM references.

import { useRef, useEffect, useState } from 'react';

// 1. Akses elemen DOM
function TextInput() {
  const inputRef = useRef(null);

  useEffect(() => {
    inputRef.current.focus(); // Auto-focus saat mount
  }, []);

  return <input ref={inputRef} placeholder="Auto focus..." />;
}

// 2. Track previous value
function usePrevious(value) {
  const ref = useRef(value);
  useEffect(() => {
    ref.current = value;
  }, [value]);
  return ref.current;
}

function Counter() {
  const [count, setCount] = useState(0);
  const prev = usePrevious(count);

  return (
    <div>
      <p>Sekarang: {count}, Sebelumnya: {prev}</p>
      <button onClick={() => setCount(c => c + 1)}>+1</button>
    </div>
  );
}

// 3. Store mutable value tanpa re-render
function Stopwatch() {
  const [isRunning, setIsRunning] = useState(false);
  const startTimeRef = useRef(null);

  const start = () => {
    startTimeRef.current = Date.now();
    setIsRunning(true);
  };
}

⚠️ Warning: Jangan gunakan ref sebagai pengganti state. Gunakan state jika perubahan harus terlihat di UI. Ref = "belakang layar". State = "di layar".

4. useMemo & useCallback — Optimasi Performa

useMemo meng-cache hasil kalkulasi agar tidak dihitung ulang setiap render. Berguna untuk operasi berat seperti filtering, sorting, atau perhitungan kompleks.

useCallback meng-cache fungsi agar referensinya stabil antar render. Berguna saat melempar fungsi sebagai prop ke child component yang di-wrap React.memo.

import { useState, useMemo, useCallback, memo } from 'react';

// useMemo: Cache kalkulasi mahal
function ProductList({ products, search }) {
  const filtered = useMemo(() => {
    console.log('Filtering...'); // Hanya log saat search berubah
    return products.filter(p =>
      p.name.toLowerCase().includes(search.toLowerCase())
    );
  }, [products, search]);

  return filtered.map(p => <div key={p.id}>{p.name}</div>);
}

// useCallback: Stabilkan fungsi untuk child
const HeavyChild = memo(function({ onClick, data }) {
  console.log('Child rendered');
  return <div onClick={onClick}>{data}</div>;
});

function Parent() {
  const [count, setCount] = useState(0);
  const [text, setText] = useState('');

  const handleClick = useCallback(() => {
    console.log('clicked');
  }, []); // Fungsi stabil — Child tidak re-render

  return (
    <div>
      <input value={text} onChange={e => setText(e.target.value)} />
      <HeavyChild onClick={handleClick} data="hello" />
      <button onClick={() => setCount(c => c + 1)}>{count}</button>
    </div>
  );
}

💡 Tip: useMemo itu seperti memo catatan — kalau pertanyaannya sama, tinggal buka memo. useCallback itu seperti menyimpan fungsi di dalam box — selama dependency sama, isinya fungsi yang sama persis. Jangan gunakan keduanya kecuali ada masalah performa nyata!

5. useContext — Akses Global State Tanpa Props Drilling

useContext memungkinkan komponen mengakses data global tanpa melempar props dari parent ke child secara berantai (props drilling).

Context terdiri dari: createContext, Provider, dan useContext.

import { createContext, useContext, useState } from 'react';

// 1. Buat Context
const ThemeContext = createContext();

// 2. Provider di root
function App() {
  const [theme, setTheme] = useState('dark');
  return (
    <ThemeContext.Provider value={{ theme, toggleTheme: () =>
      setTheme(t => t === 'dark' ? 'light' : 'dark')
    }}>
      <Header />
      <Main />
    </ThemeContext.Provider>
  );
}

// 3. Konsumsi di component manapun — tanpa props!
function Header() {
  const { theme, toggleTheme } = useContext(ThemeContext);
  return (
    <header className={theme}>
      <button onClick={toggleTheme}>
        {theme === 'dark' ? '☀️ Light' : '🌙 Dark'}
      </button>
    </header>
  );
}

function DeepChild() {
  const { theme } = useContext(ThemeContext);
  return <span style={{ color: theme === 'dark' ? '#fff' : '#000' }}>Deep child!</span>;
}

💡 Tip: useContext itu seperti pengumuman di pengeras suara. Semua orang di dalam gedung (component tree) bisa dengar tanpa harus disampaikan satu-satu (props drilling). Cocok untuk theme, auth, locale, tapi tidak untuk data yang sering berubah (gunakan state management library).

6. Kesalahan Umum Saat Menggunakan Hooks

Berikut kesalahan paling sering dilakukan developer pemula saat menggunakan hooks:

// ❌ KESALAHAN 1: Conditional hooks
function Bad({ isLoggedIn }) {
  if (isLoggedIn) {
    const [user, setUser] = useState(null); // ERROR!
  }
}
// ✅ BENAR: Selalu di top level
function Good({ isLoggedIn }) {
  const [user, setUser] = useState(null);
  return isLoggedIn ? <div>{user.name}</div> : null;
}

// ❌ KESALAHAN 2: Stale closure
function Stale() {
  const [count, setCount] = useState(0);
  useEffect(() => {
    const id = setInterval(() => {
      setCount(count + 1); // count selalu 0!
    }, 1000);
    return () => clearInterval(id);
  }, []);
}
// ✅ BENAR: Functional update
function Fixed() {
  const [count, setCount] = useState(0);
  useEffect(() => {
    const id = setInterval(() => {
      setCount(c => c + 1); // Selalu ambil nilai terbaru
    }, 1000);
    return () => clearInterval(id);
  }, []);
}

// ❌ KESALAHAN 3: Object sebagai dependency
useEffect(() => { fetchData(options); }, [options]);
// options selalu object baru → infinite loop!

// ✅ BENAR: Primitive atau useMemo
const memoOptions = useMemo(() => ({ page, limit }), [page, limit]);
useEffect(() => { fetchData(memoOptions); }, [memoOptions]);

⚠️ Warning: Aturan #1: Hooks HANYA boleh dipanggil di top level, tidak boleh di loop/condition/nested function. Aturan #2: Gunakan functional update c => c + 1 saat state baru bergantung pada state lama di useEffect. Aturan #3: Object/array selalu referensi baru — gunakan primitive atau useMemo sebagai dependency.
