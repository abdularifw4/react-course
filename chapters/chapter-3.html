01

KENAPA SIH HOOKS ITU PENTING BANGET? 🤔

Konsep inti · 8 menit baca

Oke, bayangin kamu lagi bikin aplikasi e-commerce. User klik tombol "Tambah ke Keranjang" — tapi angkanya nggak berubah. Di layar tetap nol. User klik lagi, tetap nol. User frustasi, kamu juga frustasi. 😔

Nah itu masalahnya kalau komponen kamu nggak punya state. Komponen fungsional dulu itu kayak kertas — sekali ditulis, selesai. Nggak bisa ngubah apa-apa setelah itu. Setiap kali user berinteraksi, komponen di-render ulang dari nol, tanpa ingatan sedikitpun. Kayak film 50 First Dates — bangun tidur, lupa semuanya. 😅

Sebelum React 16.8 (dirilis 2019), kalau mau punya state, kamu harus pakai class component. Nulis constructor(), this.setState(), componentDidMount(), this.handleClick.bind(this) — belajar hal-hal yang sebenernya nggak perlu buat yang baru mulai. 🤯

Hooks vs Class Component — merasa bedanya? 
// ❌ Dulu — Class Component (banyak boilerplate!)
class Counter extends React.Component {
  constructor(props) {
    super(props);
    this.state = { count: 0 };
    this.handleClick = this.handleClick.bind(this);
  }
  handleClick() {
    this.setState({ count: this.state.count + 1 });
  }
  render() {
    return <button onClick={this.handleClick}>
      Count: {this.state.count}
    </button>;
  }
}

// ✅ Sekarang — Hooks (bersih, simpel!)
function Counter() {
  const [count, setCount] = useState(0);
  return <button onClick={() => setCount(count + 1)}>
    Count: {count}
  </button>;
}

Masuklah React Hooks. Hooks itu cara React bilang ke komponen fungsional: "Hei, kamu sekarang bisa punya memori sendiri, lho!" 🧠 Dengan hooks, komponen fungsional yang tadinya cuma bisa render statis sekarang bisa punya state, lifecycle, dan semua kemampuan class component.

Analoginya gini: Hooks itu kayak upgrade gratis buat motor kamu. Dari yang awalnya cuma bisa jalan lurus (komponen tanpa state), jadi bisa belok, ngerem, ngebut, dan punya GPS (punya state, lifecycle, ref, dll). Tanpa perlu ganti motor — tetap pakai fungsi biasa yang ringan dan enak dibaca. ✨

Di bab ini, kita bakal bahas 6 hooks dasar: useState (memori), useEffect (side effect), useRef (laci rahasia), useMemo (cache), useCallback (cache fungsi), dan useContext (global state). Siap? Gaspol! 🚀

💡 Kenapa namanya "Hooks"? Karena hooks secara literal "mengaitkan" (hook into) fitur React ke komponen fungsional. useState mengaitkan fitur state, useEffect mengaitkan fitur lifecycle. Kalau ada function yang diawali use, itu artinya dia lagi "mengaitkan" sesuatu ke komponen kamu. 😎

02

USESTATE DEEP DIVE — "INGET-IN YANG INI YA!" 🧠

Fundamental · 12 menit baca

useState itu obat pertama buat komponen kamu yang amnesia. Dia kasih "ingatan" ke komponen fungsional. Jadi React bisa ngerekam: "Oke, count-nya sekarang 5. Kalau user klik lagi, jadi 6." Dan setiap kali state berubah, React otomatis re-render komponen dengan nilai terbaru. UI langsung update tanpa DOM manipulation manual. 😎

useState — Counter & Multiple States 
import { useState } from 'react';

// Sintaks dasar — destrukturisasi array
const [namaState, setNamaState] = useState(nilaiAwal);
//      ↑               ↑                ↑
//   baca data     ubah data        nilai awal

function Counter() {
  const [count, setCount] = useState(0);
  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Tambah</button>
      <button onClick={() => setCount(count - 1)}>Kurang</button>
      <button onClick={() => setCount(0)}>Reset</button>
    </div>
  );
}

// Multiple states — kayak punya banyak laci 🗄️
function Form() {
  const [nama, setNama] = useState('');
  const [email, setEmail] = useState('');
  const [submitted, setSubmitted] = useState(false);

  const handleSubmit = (e) => {
    e.preventDefault();
    setSubmitted(true);
  };

  if (submitted) return <p>Terima kasih, {nama}! 🎉</p>;

  return (
    <form onSubmit={handleSubmit}>
      <input value={nama} onChange={e => setNama(e.target.value)}
        placeholder="Nama kamu..." />
      <input value={email} onChange={e => setEmail(e.target.value)}
        placeholder="Email kamu..." />
      <button type="submit">Kirim!</button>
    </form>
  );
}

Teknik 1: Object State — WAJIB SPREAD! Kalau state kamu berbentuk objek, kamu WAJIB spread objek lamanya dulu sebelum update. Kalau nggak, field lain bakal hilang! 🚨

Object State — Hati-hati jangan lupa spread! 
// State object
const [user, setUser] = useState({ name: '', age: 0, email: '' });

// ❌ SALAH — menghapus key lain!
setUser({ name: 'Budi' }); // age dan email hilang!

// ✅ BENAR — spread dulu, baru update
setUser({ ...user, name: 'Budi' }); // age dan email masih ada

// ✅ LEBIH BAIK — functional update (paling aman)
setUser(prev => ({ ...prev, name: 'Budi' }));

// Praktik: Form dengan dynamic key
const handleChange = (e) => {
  const { name, value } = e.target;
  setUser(prev => ({ ...prev, [name]: value }));
};

Teknik 2: Functional Update. Kalau state baru bergantung pada state lama, WAJIB pakai functional update: setCount(prev => prev + 1). Kenapa? Karena React bisa nge-batch update state, dan nilai count mungkin udah "stale" pas eksekusi. Functional update menjamin kamu selalu pakai nilai terbaru. 🧠

Functional Update — kapan harus pakai? 
// ❌ SALAH — state bisa stuck karena stale closure
const addThreeWrong = () => {
  setCount(count + 1); // pakai count yang sama 3x!
  setCount(count + 1); // hasilnya cuma +1, bukan +3
  setCount(count + 1);
};

// ✅ BENAR — functional update selalu pakai nilai terbaru
const addThreeRight = () => {
  setCount(prev => prev + 1); // chain update
  setCount(prev => prev + 1); // masing-masing pakai latest
  setCount(prev => prev + 1); // hasilnya: +3 ✅
};

Teknik 3: Lazy Initialization. Kalau initial state-nya butuh kalkulasi mahal (misal parse data besar, hitung dari localStorage), kamu bisa pass function ke useState. Function ini cuma jalan sekali saat mount — bukan setiap render! 🎯

Lazy Initialization — cuma jalan sekali! 
// ❌ BERAT — expensiveCalculation() jalan SETIAP render!
const [data, setData] = useState(expensiveCalculation());

// ✅ RINGAN — cuma jalan sekali saat mount!
const [data, setData] = useState(() => {
  return expensiveCalculation();
});

// Contoh nyata: baca dari localStorage
const [theme, setTheme] = useState(() => {
  return localStorage.getItem('theme') || 'light';
});

💡 Mindset yang bener: Bayangin useState itu kayak variabel yang bisa update tampilan. Bedanya sama variabel biasa, kalau kamu ubah useState, React otomatis gambar ulang komponen. Kalau variabel biasa? Diubah di belakang layar, tapi di layar tetap yang lama. Jadi kalau mau perubahannya keliatan di UI, pakai useState! 🤯

⚠️ Yang sering salah: Jangan langsung mutate state! count++ atau array.push() nggak bakal kerja. React butuh immutable update — selalu pakai setter function yang disediakan useState!

03

USEEFFECT MASTERCLASS — "REACT, LAKUKAN INI!" ⚡

Praktik · 12 menit baca

Okay, kamu udah bisa simpan data pakai useState. Tapi gimana cara fetch data dari API? Atau set timer? Atau langganan event listener? Di dunia class component dulu, kamu pake lifecycle methods kayak componentDidMount, componentDidUpdate, componentWillUnmount. Tiga method buat tiga fase berbeda. Ribet! 🤯

useEffect gabungin semua itu jadi SATU hook. Dia bilang ke React: "Hei, setelah kamu selesai render, lakukanin ini ya." ⚡

useEffect — Fetch data dengan loading state 
import { useState, useEffect } from 'react';

// Skenario 1: Fetch data saat component pertama kali muncul
function UserProfile({ userId }) {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    setLoading(true);
    fetch(`/api/users/${userId}`)
      .then(res => res.json())
      .then(data => {
        setUser(data);
        setLoading(false);
      });
  }, [userId]); // ← dependency array penting!

  if (loading) return <p>Loading... ⏳</p>;
  return <p>Halo, {user.name}! 👋</p>;
}

Perhatikan [userId] di akhir useEffect. Itu namanya dependency array — dan ini KUNCI buat paham useEffect. Dependency array menentukan kapan effect kamu dijalankan ulang.

Cleanup — JANGAN LUPA INI! Kalau effect kamu punya sesuatu yang perlu "dibersihkan" (timer, event listener, WebSocket), kamu WAJIB return cleanup function. Tanpa cleanup, timer bakal jalan terus walau komponen udah hilang. Itu namanya memory leak — bikin app lambat kayak siput 🐌.

useEffect — Cleanup & Multiple Effects 
// Effect dengan cleanup (timer)
function Timer() {
  const [seconds, setSeconds] = useState(0);

  useEffect(() => {
    const timer = setInterval(() => {
      setSeconds(s => s + 1);
    }, 1000);
    // Cleanup: STOP timer saat component hilang
    return () => clearInterval(timer);
  }, []); // [] = jalan sekali saat mount

  return <p>{seconds} detik berlalu ⏱️</p>;
}

// Bisa punya banyak effect sekaligus!
function Dashboard() {
  const [data, setData] = useState([]);
  const [theme, setTheme] = useState('dark');

  // Effect 1: Fetch data saat mount
  useEffect(() => {
    fetch('/api/data').then(r => r.json()).then(setData);
  }, []);

  // Effect 2: Ubah class body setiap kali theme berubah
  useEffect(() => {
    document.body.className = theme;
  }, [theme]);
}

Cheat Sheet Dependency Array: Ini tiga pola dependency array yang wajib kamu hafal:

Dependency Array — 3 pola wajib hafal 
// 🚫 TANPA ARRAY → jalan SETIAP render (hati-hati infinite loop!)
useEffect(() => { console.log('jalan setiap render'); });

// 📦 [] (kosong) → jalan sekali saat mount
useEffect(() => {
  fetchData();
}, []); // kayak componentDidMount

// 🎯 [dep1, dep2] → jalan saat dep berubah
useEffect(() => {
  fetch(`/api/users/${userId}`);
}, [userId]); // kayak componentDidMount + componentDidUpdate

// ❌ HATI-HATI: infinite loop!
useEffect(() => {
  setCount(count + 1); // trigger re-render → effect jalan lagi → infinity!
}); // tidak ada dependency array!

💡 Cheat sheet dependency array:

  • 🚫 Tanpa array → Jalan SETIAP render. Bisa infinite loop!
  • 📦 [] (kosong) → Jalan sekali saat mount. Kayak componentDidMount
  • 🎯 [dep1, dep2] → Jalan saat dep berubah. Kayak componentDidUpdate spesifik

⚠️ JANGAN PERNAH lupa dependency array! Kalau kamu tulis useEffect(() => { ... }) tanpa array di akhir, dia bakal jalan setiap render. Artinya fetch API setiap detik. Setiap. Detik. SERIUS. 🫢

04

USEREF — "LACI RAHASIA TERSEMBUNYI" 🔮

Fundamental · 10 menit baca

Pernah mau auto-focus ke input field saat halaman kebuka? Atau butuh nge-track nilai sebelumnya? Atau simpan timer ID? Nah kalau pakai useState buat hal-hal itu, kamu bakal kena re-render yang nggak perlu. Re-render yang nggak perlu itu ibarat kamu disuruh tulis ulang seluruh dokumen cuma karena salah satu huruf. 😕

useRef itu kayak laci rahasia. Kamu simpan apa aja di sana, nggak ada yang tau, dan nggak memicu re-render. Cocok banget buat hal-hal yang "di belakang layar" — akses DOM, simpan mutable value, track previous value. 🗄️

useRef — Akses DOM & Previous Value 
import { useRef, useEffect, useState } from 'react';

// Use case 1: Auto-focus input (paling populer)
function TextInput() {
  const inputRef = useRef(null);

  useEffect(() => {
    inputRef.current.focus(); // Langsung akses DOM!
  }, []);

  return <input ref={inputRef} placeholder="Langsung fokus, bro..." />;
}

// Use case 2: Track previous value
function usePrevious(value) {
  const ref = useRef(value);
  useEffect(() => {
    ref.current = value; // Simpan setelah render
  }, [value]);
  return ref.current;
}

function Counter() {
  const [count, setCount] = useState(0);
  const prevCount = usePrevious(count);

  return (
    <div>
      <p>Sekarang: {count} | Sebelumnya: {prevCount}</p>
      <button onClick={() => setCount(c => c + 1)}>+1</button>
    </div>
  );
}

// Use case 3: Simpan interval ID (mutable storage)
function Stopwatch() {
  const [seconds, setSeconds] = useState(0);
  const intervalRef = useRef(null);

  useEffect(() => {
    intervalRef.current = setInterval(() => {
      setSeconds(s => s + 1);
    }, 1000);
    return () => clearInterval(intervalRef.current);
  }, []);

  const stop = () => clearInterval(intervalRef.current);

  return <div>
    <p>{seconds}s</p>
    <button onClick={stop}>Stop</button>
  </div>;
}

useState vs useRef — Kapan pakai apa? Ini pertanyaan yang sering muncul. Perbedaan utamanya gampang: kalau perubahannya harus keliatan di UI (trigger re-render), pakai useState. Kalau cuma buat internal tracking yang nggak perlu keliatan di layar, pakai useRef. Ref itu backend, state itu frontend. Jangan kebalik! 🔀

useState vs useRef — perbedaan kritis 
// useState: perubahan → RE-RENDER
const [count, setCount] = useState(0);
setCount(5); // UI update, component re-render

// useRef: perubahan → TANPA RE-RENDER
const countRef = useRef(0);
countRef.current = 5; // UI TIDAK update, silent change

// Contoh: render count tanpa re-render?
// Pakai useState! Karena kamu MAU angkanya keliatan.

// Contoh: simpan scroll position sementara?
// Pakai useRef! Nggak perlu keliatan di UI saat disimpan.

⚠️ Aturan emas useRef: ref.current bisa kamu ubah kapan aja, tapi kalau perubahannya harus keliatan di UI — pakai useState, BUKAN useRef. Ref itu backend, state itu frontend. Jangan kebalik! 🔀

USEMEMO & USECALLBACK — "JANGAN KERJA DUA KALI!" 📦

Performance · 10 menit baca

Bayangin kamu punya list 10.000 produk dan tiap kali user ngetik di search bar, kamu filter ulang SEMUA produk. Nah kalau proses filtering itu berat, UI kamu bisa lag. User klik, tunggu... klik lagi, tunggu... 😴

useMemo dan useCallback itu kayak cache di browser. Udah dihitung/dibuat sekali, kalau inputnya sama, tinggal pakai hasil yang lama. Done. ✅

useMemo — Cache hasil kalkulasi mahal 
import { useState, useMemo } from 'react';

    function ProductList({ products, search }) {
    // useMemo: cuma re-filter kalau products atau search berubah
    const filtered = useMemo(() => {
     console.log('Filtering 10.000 produk...');
     return products.filter(p =>
       p.name.toLowerCase().includes(search.toLowerCase())
     );
    }, [products, search]);

    return (
     <ul>
       {filtered.map(p => <li key={p.id}>{p.name}</li>)}
     </ul>
    );
    }
useCallback + memo — Cache fungsi biar child nggak re-render 
import { useState, useCallback, memo } from 'react';

    // memo: cegah re-render kalau props nggak berubah
    const HeavyChild = memo(function({ onClick, data }) {
    console.log('Child di-render lagi nih...');
    return <div onClick={onClick}>{data}</div>;
    });

    function Parent() {
    const [count, setCount] = useState(0);
    const [text, setText] = useState('');

    // useCallback: fungsi stabil selama deps nggak berubah
    const handleClick = useCallback(() => {
     console.log('clicked!');
    }, []); // Fungsi stabil → HeavyChild nggak re-render

    return (
     <div>
       <input value={text} onChange={e => setText(e.target.value)} />
       <HeavyChild onClick={handleClick} data="hello" />
       <button onClick={() => setCount(c => c + 1)}>Count: {count}</button>
     </div>
    );
    }

Kapan pakai apa? useMemo buat cache nilai/kalkulasi. useCallback buat cache fungsi supaya referensinya stabil antar render. Biasanya pakai bareng memo() pada child component biar re-render-nya benar-benar dicegah.

💡 Jangan kebablasan! useMemo dan useCallback bukan berarti harus dipake di SEMUA tempat. Keduanya juga punya "biaya" — React harus nyimpen hasilnya di memori. Pakai kalau memang ada masalah performa nyata, bukan sekadar "kayaknya lebih baik". Premature optimization itu jahat! 😈

USECONTEXT — "PENGUMUMAN UNTUK SEMUA" 📢

Global State · 10 menit baca

Pernah ngalamin props drilling? Jadi gini: kamu punya data theme di App, terus di-lempar ke Header → Navbar → MenuButton. Tiga component cuma buat nganterin SATU data. Rasanya kayak jadi kurir paket, tapi cuma bawa satu amplop 📦

useContext itu solusinya. Dia kayak pengeras suara di sekolah. Ketua kelas (Provider) bilang sekali, semua murid (component) denger. Nggak perlu disampaikan satu-satu. 📢

useContext — Theme toggle tanpa props drilling 
import { createContext, useContext, useState } from 'react';

    // Step 1: Buat context
    const ThemeContext = createContext();

    // Step 2: Bungkus app dengan Provider
    function App() {
    const [theme, setTheme] = useState('dark');

    return (
     <ThemeContext.Provider value={{
       theme,
       toggleTheme: () => setTheme(t => t === 'dark' ? 'light' : 'dark')
     }}>
       <Header />
       <Main />
     </ThemeContext.Provider>
    );
    }

    // Step 3: Pakai di MANA SAJA — tanpa props!
    function Header() {
    const { theme, toggleTheme } = useContext(ThemeContext);
    return (
     <header className={theme}>
       <button onClick={toggleTheme}>
         {theme === 'dark' ? '☀️ Light Mode' : '🌙 Dark Mode'}
       </button>
     </header>
    );
    }

    // Bahkan di child yang paling dalam
    function DeepChild() {
    const { theme } = useContext(ThemeContext);
    return <span style={{ color: theme === 'dark' ? '#fff' : '#000' }}>
     Aku di sini tapi tetap bisa akses theme! 🎉
    </span>;
    }
useContext — Auth example 
import { createContext, useContext, useState } from 'react';

    const AuthContext = createContext();

    function AuthProvider({ children }) {
    const [user, setUser] = useState(null);

    const login = (email, password) => {
     // Simulasi login
     setUser({ email, name: 'Arif' });
    };

    const logout = () => setUser(null);

    return (
     <AuthContext.Provider value={{ user, login, logout }}>
       {children}
     </AuthContext.Provider>
    );
    }

    // Pakai di mana saja
    function Navbar() {
    const { user, logout } = useContext(AuthContext);
    return user
     ? <div>Halo, {user.name}! <button onClick={logout}>Logout</button></div>
     : <div>Silakan login</div>;
    }

💡 Kapan pakai useContext? Cocok buat data yang "global" dan jarang berubah: theme 🎨, auth user 🔐, locale/language 🌍. Tapi JANGAN pakai buat data yang sering banget berubah tiap detik — nanti semua component yang konsumsi bakal re-render terus. Buat itu, pakai state management library kayak Zustand atau Redux.

07

COMMON MISTAKES & BEST PRACTICES 🤦

Best Practices · 10 menit baca

Ini bagian yang paling berharga. Error yang bakal kamu temuin di dunia nyata, dan cara ngefix-nya. Belajar dari kesalahan orang lain itu lebih murah daripada ngalamin sendiri 💸

Kesalahan 1: Hooks di dalam condition/loop. React butuh hooks selalu dipanggil dalam URUTAN YANG SAMA setiap render. Kalau kadang ada, kadang nggak — React bingung! Hooks harus selalu di top level component. 🚨

Kesalahan 2: Stale Closure. Ini yang PALING bikin pusing. useEffect "nyimpan" nilai dari render pertama dan nggak update. Solusinya: selalu pakai functional update di dalam useEffect. 🧠

Kesalahan 3: Object/Array sebagai dependency. Setiap render, object BARU dibuat di JavaScript. Referensi berbeda → React anggap berubah → infinite loop! Solusi: pakai primitive atau useMemo. 🔄

3 Kesalahan Fatal & Cara Fix-nya 
// ❌ KESALAHAN 1: Hooks di dalam condition
     function Bad({ isLoggedIn }) {
     if (isLoggedIn) {
     const [user, setUser] = useState(null); // 💥 BOOM!
     }
     }

     // ✅ FIX: Selalu di top level
     function Good({ isLoggedIn }) {
     const [user, setUser] = useState(null);
     if (!isLoggedIn) return null;
     return <div>{user.name}</div>;
     }

     // ❌ KESALAHAN 2: Stale closure di useEffect
     function Stale() {
     const [count, setCount] = useState(0);
     useEffect(() => {
     const id = setInterval(() => {
       setCount(count + 1); // count selalu 0! 😭
     }, 1000);
     return () => clearInterval(id);
     }, []);
     }

     // ✅ FIX: Pakai functional update!
     function Fixed() {
     const [count, setCount] = useState(0);
     useEffect(() => {
     const id = setInterval(() => {
       setCount(c => c + 1); // Selalu ambil nilai TERBARU 🎯
     }, 1000);
     return () => clearInterval(id);
     }, []);
     }

     // ❌ KESALAHAN 3: Object sebagai dependency
     useEffect(() => { fetchData(options); }, [options]);
     // options = { page: 1, limit: 10 } → referensi BARU setiap render!

     // ✅ FIX: Pakai primitive atau useMemo
     const memoOptions = useMemo(() => ({ page, limit }), [page, limit]);
     useEffect(() => { fetchData(memoOptions); }, [memoOptions]);
3 Aturan Emas Hooks 
// ❌ 1. Lupa dependency array
     useEffect(() => { fetchData(); });
     // jalan setiap render! Performance disaster!

     // ✅ Fix:
     useEffect(() => { fetchData(); }, []); // cuma sekali

     // ❌ 2. Update state berdasarkan state lama
     const [count, setCount] = useState(0);
     setCount(count + 1); // bisa stale kalau dipanggil cepat
     setCount(count + 1); // count masih nilai lama!

     // ✅ Fix: gunakan functional update
     setCount(prev => prev + 1);
     setCount(prev => prev + 1); // pasti +2

     // ❌ 3. Panggil hook di dalam conditional
     if (showCount) {
     const [count, setCount] = useState(0); // BREAKS RULES!
     }

     // ✅ Fix: hook selalu di top level
     const [count, setCount] = useState(0);
     if (showCount) { /* pakai count di sini */ }

⚠️ 3 Aturan Emas Hooks:

  1. Top level only! Nggak boleh di dalam if, loop, atau nested function.
  2. Functional update c => c + 1 kalau state baru bergantung di state lama.
  3. Primitive dependency! Object/array = referensi baru setiap render. Pakai useMemo atau pecah jadi primitive.

MINI PROJECT: STOPWATCH APP ⏱️

Praktik · 15 menit

Oke, teori udah cukup! Sekarang kita gabungkan useState + useEffect + useRef dalam satu mini project nyata: Stopwatch App. Ini project sempurna karena dia pakai tiga hooks paling penting sekaligus. 🎉

Plan: Stopwatch punya tombol Start/Pause/Reset, menampilkan waktu yang berjalan, dan menyimpan lap times. Kita pakai useState buat waktu dan status, useEffect buat interval timer, dan useRef buat simpan interval ID. 🧩

Stopwatch App — Full Code! 
import { useState, useEffect, useRef } from 'react';

function Stopwatch() {
  // State buat waktu dan status
  const [time, setTime] = useState(0); // dalam milidetik
  const [isRunning, setIsRunning] = useState(false);
  const [laps, setLaps] = useState([]);

  // Ref buat simpan interval ID (nggak trigger re-render)
  const intervalRef = useRef(null);
  const startTimeRef = useRef(0);

  // useEffect: jalankan/pause interval
  useEffect(() => {
    if (isRunning) {
      startTimeRef.current = Date.now() - time;
      intervalRef.current = setInterval(() => {
        setTime(Date.now() - startTimeRef.current);
      }, 10);
    } else {
      clearInterval(intervalRef.current);
    }
    return () => clearInterval(intervalRef.current);
  }, [isRunning]);

  // Helper: format milidetik jadi MM:SS.ms
  const formatTime = (ms) => {
    const minutes = Math.floor(ms / 60000);
    const seconds = Math.floor((ms % 60000) / 1000);
    const millis = Math.floor((ms % 1000) / 10);
    return `${String(minutes).padStart(2,'0')}:${String(seconds).padStart(2,'0')}.${String(millis).padStart(2,'0')}`;
  };

  const handleStartPause = () => setIsRunning(!isRunning);
  const handleReset = () => { setIsRunning(false); setTime(0); setLaps([]); };
  const handleLap = () => {
    if (isRunning) setLaps(prev => [...prev, time]);
  };

  return (
    <div className="p-6 max-w-sm mx-auto bg-gray-900 rounded-2xl text-white">
      <div className="text-center mb-6">
        <p className="text-5xl font-mono tracking-wider">{formatTime(time)}</p>
      </div>
      <div className="flex gap-3 justify-center mb-6">
        <button onClick={handleStartPause}
          className="px-6 py-2 rounded-xl font-medium transition-colors"
          style={{ background: isRunning ? '#ef4444' : '#22c55e' }}>
          {isRunning ? 'Pause' : 'Start'}
        </button>
        <button onClick={handleLap} disabled={!isRunning}
          className="px-6 py-2 rounded-xl font-medium bg-gray-700 disabled:opacity-40">
          Lap
        </button>
        <button onClick={handleReset}
          className="px-6 py-2 rounded-xl font-medium bg-gray-700">
          Reset
        </button>
      </div>
      {laps.map((lap, i) => (
        <div key={i} className="flex justify-between text-sm px-3 py-1 bg-gray-800 rounded mb-1">
          <span>Lap {i + 1}</span>
          <span className="font-mono">{formatTime(lap)}</span>
        </div>
      ))}
    </div>
  );
}

export default Stopwatch;

Perhatikan bagaimana ketiga hooks bekerja sama: useState ngatur data yang perlu keliatan di UI (waktu, status, laps), useEffect ngatur side effect (timer interval), dan useRef nyimpen nilai yang nggak perlu trigger re-render (interval ID, start time). 💪
