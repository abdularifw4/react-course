🤔 KENAPA SIH HOOKS ITU PENTING BANGET?

Oke, bayangin kamu lagi bikin aplikasi e-commerce. User klik tombol “Tambah ke Keranjang” — tapi angkanya nggak berubah. Di halaman tetap nol. 😔

Nah itu masalahnya kalau komponen kamu nggak punya state. Komponen fungsional dulu itu kayak kertas — sekali ditulis, selesai. Nggak bisa ngubah apa-apa setelah itu.

Masuklah React Hooks di React 16.8 (2019). Hooks itu cara React bilang ke komponen fungsional: “Hei, kamu sekarang bisa punya memori sendiri, lho!” 🧠

Sebelum hooks, kalau mau punya state, kamu harus pakai class component. Yang artinya nulis constructor(), this.setState(), componentDidMount() — belajar hal-hal yang sebenernya nggak perlu buat yang baru mulai.

Sekarang? Cukup fungsi biasa + hooks. Lebih simpel, lebih gampang dibaca, dan—yang paling penting—lebih fun. ✨

💡 Tip: Analoginya gini: Hooks itu kayak upgrade gratis buat motor kamu. Dari yang awalnya cuma bisa jalan lurus (komponen tanpa state), jadi bisa belok, ngerem, dan ngebut (punya state, lifecycle, dll). Tanpa perlu ganti motor (ke class component).

1️⃣ USESTATE — “INGET-IN YANG INI YA, REACT!”

Pernah ngalamin ini? Kamu bikin tombol counter, tapi angkanya nggak pernah nambah. Kamu udah pasang onClick, udah nulis count++... tapi UI-nya tetap 0. 🤯

Itu karena komponen fungsional nggak punya memori. Setiap kali di-render, dia lupa sama sebelumnya. Kayak film 50 First Dates — bangun, lupa semuanya. 😅

useState itu obatnya. Dia kasih “ingatan” ke komponen kamu. Jadi React bisa ngerekam: “Oke, count-nya sekarang 5. Kalau user klik lagi, jadi 6.”

// Sintaks dasar — destrukturisasi array
const [namaState, setNamaState] = useState(nilaiAwal);
//      ↑               ↑                ↑
//   baca data     ubah data        nilai awal

// Contoh paling klasik: Counter 🎯
import { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  return (
    

      
Count: {count}

      
      
      
    

  );
}

Gampang kan? count itu nilainya sekarang, setCount itu cara ngubahnya. Setiap kali kamu panggil setCount, React bakal otomatis re-render komponen dengan nilai baru.

Tapi tunggu—kamu nggak cuma bisa punya SATU state. Satu komponen bisa punya banyak useState, kayak gini:

// Multiple states — kayak punya banyak laci 🗄️
function Form() {
  const [nama, setNama] = useState('');
  const [email, setEmail] = useState('');
  const [submitted, setSubmitted] = useState(false);

  const handleSubmit = (e) => {
    e.preventDefault();
    setSubmitted(true);
  };

  if (submitted) {
    return 
Terima kasih, {nama}! 🎉
;
  }

  return (
    

       setNama(e.target.value)}
        placeholder="Nama kamu..."
      />
       setEmail(e.target.value)}
        placeholder="Email kamu..."
      />

💡 Tip: Mindset yang bener tentang useState: Bayangin useState itu kayak variabel yang bisa update tampilan. Bedanya sama variabel biasa, kalau kamu ubah useState, React otomatis gambar ulang (re-render) komponen. Kalau variabel biasa? Diubah di belakang layar, tapi di layar tetap yang lama. Oh ternyata gitu! 🤯

⚠️ Warning: Yang sering salah: Jangan langsung mutate state! count++ atau array.push() nggak bakal kerja. React butuh immutable update — selalu pakai setter function yang udah disediakan useState.

2️⃣ USEEFFECT — “REACT, LAKUKAN INI SETELAH RENDER”

Okay, kamu udah bisa simpan data pakai useState. Tapi sekarang gimana cara fetch data dari API? Atau set timer? Atau langganan event?

Di dunia class component dulu, kamu pake lifecycle methods kayak componentDidMount, componentDidUpdate, dan componentWillUnmount. Tiga method buat tiga fase berbeda. Ribet, kan?

useEffect gabungin semua itu jadi SATU hook. Dia bilang ke React: “Hei, setelah kamu selesai render, lakukanin ini ya.” 🫥

import { useState, useEffect } from 'react';

// Skenario 1: Fetch data saat component pertama kali muncul
function UserProfile({ userId }) {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    // Ini jalan SETELAH render pertama
    setLoading(true);
    fetch(`/api/users/${userId}`)
      .then(res => res.json())
      .then(data => {
        setUser(data);
        setLoading(false);
      });
  }, [userId]); // ← INI PENTING! Bahas di bawah ya

  if (loading) return 
Loading... ⏳
;
  return 
Halo, {user.name}! 👋
;
}

Perhatikan yang di atas itu [userId] di akhir useEffect. Itu namanya dependency array — dan ini adalah KUNCI buat paham useEffect. Beneran, kalau nggak paham ini, kamu bakal kena bug yang bikin pusing 🤯.

// Skenario 2: Cleanup — JANGAN LUPA INI!
function Timer() {
  const [seconds, setSeconds] = useState(0);

  useEffect(() => {
    // Mulai timer
    const timer = setInterval(() => {
      setSeconds(s => s + 1);
    }, 1000);

    // Cleanup: STOP timer saat component hilang
    return () => clearInterval(timer);
  }, []); // [] = jalan sekali, cleanup saat unmount

  return 
{seconds} detik berlalu ⏱️
;
}

Nah, cleanup itu penting banget. Tanpa cleanup, timer kamu bakal jalan terus walau komponennya udah nggak ada. Itu namanya memory leak — dan itu bikin app kamu lambat kayak siput 🐌.

// Skenario 3: Banyak effect sekaligus
function Dashboard() {
  const [data, setData] = useState([]);
  const [theme, setTheme] = useState('dark');

  // Effect 1: Fetch data saat mount
  useEffect(() => {
    fetch('/api/data')
      .then(r => r.json())
      .then(setData);
  }, []);

  // Effect 2: Ubah class body setiap kali theme berubah
  useEffect(() => {
    document.body.className = theme;
  }, [theme]);

  // Boleh punya effect sebanyak yang kamu mau!
  // Masing-masing punya tugas sendiri-sendiri 👌
}

💡 Cheat sheet dependency array:

  • 🚫 Tanpa array → Jalan SETIAP render. Hati-hati, bisa infinite loop!
  • 📦 [] (kosong) → Jalan sekali saja saat mount. Kayak componentDidMount
  • 🎯 [dep1, dep2] → Jalan saat dep berubah. Kayak componentDidUpdate yang spesifik

⚠️ Warning: JANGAN PERNAH lupa dependency array! Kalau kamu tulis useEffect(() => { ... }) tanpa array di akhir, dia bakal jalan setiap render. Itu artinya fetch API setiap detik. Setiap detik. SERIUS. 🫢

3️⃣ USEREF — “RAHASIA YANG TERSEMBUNYI”

Pernah mau auto-focus ke input field saat halaman kebuka? Atau butuh nge-track nilai sebelumnya? Atau simpan timer ID?

Nah kalau pakai useState buat hal-hal itu, kamu bakal kena re-render yang nggak perlu. Dan re-render yang nggak perlu itu... ibarat kamu disuruh tulis ulang seluruh dokumen cuma karena salah satu huruf. 😕

useRef itu kayak laci rahasia. Kamu simpan apa aja di sana, nggak ada yang tau, dan nggak memicu re-render. Cocok banget buat hal-hal yang “di belakang layar”.

import { useRef, useEffect, useState } from 'react';

// 1. Auto-focus input — ini use case paling populer
function TextInput() {
  const inputRef = useRef(null);

  useEffect(() => {
    inputRef.current.focus(); // Langsung akses DOM!
  }, []);

  return (
    

      
    

  );
}

// 2. Track nilai sebelumnya
function usePrevious(value) {
  const ref = useRef(value);
  useEffect(() => {
    ref.current = value; // Simpan setelah render
  }, [value]);
  return ref.current;
}

function Counter() {
  const [count, setCount] = useState(0);
  const prevCount = usePrevious(count);

  return (
    

      
Sekarang: {count} | Sebelumnya: {prevCount}

      
    

  );
}

⚠️ Warning: Aturan emas useRef: ref.current bisa kamu ubah kapan aja, tapi kalau kamu butuh perubahannya keliatan di UI — pakai useState, BUKAN useRef. Ref itu backend, state itu frontend. Jangan kebalik! 🔀

4️⃣ USEMEMO & USECALLBACK — “JANGAN KERJA DUA KALI KALAU NGGAK PERLU”

Nah useMemo dan useCallback itu kayak cache di browser. Udah dihitung/dibuat sekali, kalau inputnya sama, tinggal pakai hasil yang lama. Done. ✅

5️⃣ USECONTEXT — “PENGUMUMAN UNTUK SEMUA ORANG”

Pernah ngalamin props drilling? Jadi gini: kamu punya data theme di App, terus di-lempar ke Header → Navbar → MenuButton. Tiga component cuma buat nganterin SATU data. Rasanya kayak jadi kurir paket, tapi cuma bawa satu amplop 📦

useContext itu solusinya. Dia kayak pengeras suara di sekolah. Ketua kelas (Provider) bilang sekali, semua murid (component) denger. Nggak perlu disampaikan satu-satu. 📢

6️⃣ KESALAHAN YANG BIKIN “AH, TERNYATA!” 🤦

Ini dia bagian yang paling berharga. Error yang bakal kamu temuin di dunia nyata, dan cara ngefix-nya. Belajar dari kesalahan orang lain itu lebih murah daripada ngalamin sendiri 💸

// ❌ KESALAHAN 1: Hooks di dalam condition/loop
function Bad({ isLoggedIn }) {
  if (isLoggedIn) {
    const [user, setUser] = useState(null); // 💥 BOOM! Error!
  }
}
// Kenapa? React butuh hooks selalu dipanggil
// dalam URUTAN YANG SAMA setiap render.
// Kalau kadang ada, kadang nggak... React bingung!

// ✅ BENAR: Selalu di top level component
function Good({ isLoggedIn }) {
  const [user, setUser] = useState(null);
  if (!isLoggedIn) return null; // Conditional di bawah hooks
  return 
{user.name}
;
}

// ❌ KESALAHAN 2: Stale closure — yang PALING bikin pusing
function Stale() {
  const [count, setCount] = useState(0);
  useEffect(() => {
    const id = setInterval(() => {
      setCount(count + 1); // count selalu 0! 😭
    }, 1000);
    return () => clearInterval(id);
  }, []);
  // Kenapa? Karena useEffect “nyimpan” count = 0 dari render pertama.
  // Dia nggak update. Itu namanya stale closure.

// ✅ BENAR: Pakai functional update!
function Fixed() {
  const [count, setCount] = useState(0);
  useEffect(() => {
    const id = setInterval(() => {
      setCount(c => c + 1); // Selalu ambil nilai TERBARU 🎯
    }, 1000);
    return () => clearInterval(id);
  }, []);
}

// ❌ KESALAHAN 3: Object sebagai dependency
useEffect(() => { fetchData(options); }, [options]);
// options = { page: 1, limit: 10 }
// Masalahnya: setiap render, object BARU dibuat!
// Referensi berbeda → React anggap berubah → infinite loop! 🔄

// ✅ BENAR: Pakai primitive atau useMemo
const memoOptions = useMemo(() => ({ page, limit }), [page, limit]);
useEffect(() => { fetchData(memoOptions); }, [memoOptions]);

⚠️ 3 Aturan Emas Hooks:

  1. Top level only! Nggak boleh di dalam if, loop, atau nested function.
  2. Functional update c => c + 1 kalau state baru bergantung di state lama di useEffect.
  3. Primitive dependency! Object/array = referensi baru setiap render. Pakai useMemo atau pecah jadi primitive.

📝 RECAP — YANG KAMU UDAH PELAJARI

Oke, kita udah bahas 6 hooks dasar React. Ini rangkumannya biar kamu nggak lupa:

  • 🧠 useState — Kasih “ingatan” ke komponen. Data yang berubah = re-render.
  • ⚡ useEffect — Lakukan side effect setelah render. Fetch data, timer, subscription.
  • 🔮 useRef — Laci rahasia. Akses DOM, simpan mutable value, tanpa re-render.
  • 📦 useMemo — Cache hasil kalkulasi mahal. Nggak perlu hitung ulang kalau inputnya sama.
  • 🪚 useCallback — Cache fungsi biar referensinya stabil. Anak-anak (child component) senang.
  • 📢 useContext — Pengumuman global. Akses data tanpa props drilling berantai.

💡 Tip: Nggak perlu hafal semua sekarang. Yang paling penting kamu mahir dulu itu useState dan useEffect — itu duo yang bakal kamu pakai 90% waktu. Sisanya pelajari pelan-pelan sambil ngoding proyek nyata. Trust the process! 💪
