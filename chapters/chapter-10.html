Performance & Best Practices

1. React.memo & useMemo — Hindari Re-render Tidak Perlu

React default-nya me-re-render semua child saat parent berubah. React.memo dan useMemo membantu menghentikan re-render yang tidak perlu.

import { memo, useMemo, useState } from 'react';

// React.memo: skip re-render jika props sama
const UserCard = memo(function UserCard({ name, avatar }) {
  console.log('UserCard rendered');
  return (
    <div className='flex items-center gap-3'>
      <img src={avatar} alt={name} className='w-10 h-10 rounded-full' />
      <span>{name}</span>
    </div>
  );
});

function UserList() {
  const [search, setSearch] = useState('');
  const users = useMemo(() => getFilteredUsers(search), [search]);

  return (
    <div>
      <input value={search} onChange={e => setSearch(e.target.value)} />
      {users.map(u => (
        <UserCard key={u.id} name={u.name} avatar={u.avatar} />
      ))}
    </div>
  );
}

// Custom comparison function
const ExpensiveCard = memo(
  function Card({ data }) { return <div>{data.title}</div>; },
  (prev, next) => prev.data.id === next.data.id // Custom equality
);

💡 Tip: React.memo itu seperti skip button di video. Kalau isinya sama (props tidak berubah), skip saja — jangan re-render. Tapi jangan memo semua! Hanya komponen yang benar-benar mahal untuk render.

2. Code Splitting & Lazy Loading

Code splitting memecah bundle JavaScript menjadi chunk kecil yang dimuat sesuai kebutuhan. Ini drastis mengurangi initial load time.

import { lazy, Suspense } from 'react';

// Lazy load route-level components
const Dashboard = lazy(() => import('./pages/Dashboard'));
const Settings = lazy(() => import('./pages/Settings'));
const Analytics = lazy(() => import('./pages/Analytics'));

function App() {
  return (
    <Suspense fallback={
      <div className='flex items-center justify-center h-screen'>
        <div className='animate-spin w-8 h-8 border-2 border-accent border-t-transparent rounded-full' />
        <span className='ml-3'>Loading...</span>
      </div>
    }>
      <Routes>
        <Route path='/' element={<Dashboard />} />
        <Route path='/settings' element={<Settings />} />
        <Route path='/analytics' element={<Analytics />} />
      </Routes>
    </Suspense>
  );
}

// Preload on hover (advanced)
function NavItem() {
  const handleMouseEnter = () => {
    // Mulai load sebelum user klik
    import('./pages/Analytics');
  };
  return <div onMouseEnter={handleMouseEnter}>Analytics</div>;
}

💡 Tip: Code splitting itu seperti memecah buku jadi beberapa bab. Kamu nggak perlu bawa semua bab sekaligus — cukup bawa bab yang sedang dibaca. Ini mengurangi ukuran initial load drastis.

3. Virtualization dengan react-window

Virtualization hanya merender item yang terlihat di viewport. Untuk list dengan ratusan atau ribuan item, ini bisa mengubah performa dari lambat jadi instant.

import { FixedSizeList as List } from 'react-window';

// 10,000 items — tapi hanya render yang terlihat!
function BigList({ items }) {
  const Row = ({ index, style }) => (
    <div style={style} className='flex items-center px-4 border-b border-dark-600'>
      <span className='text-gray-400'>{index + 1}.</span>
      <span className='ml-3'>{items[index].name}</span>
      <span className='ml-auto text-accent'>Rp{items[index].price.toLocaleString()}</span>
    </div>
  );

  return (
    <List
      height={600}         // Viewport height
      itemCount={items.length}
      itemSize={56}        // Height per item
      width='100%'
    >
      {Row}
    </List>
  );
}

// Dengan 10,000 items:
// Tanpa virtualization: render 10,000 DOM nodes = LAMBAT
// Dengan virtualization: render ~15 DOM nodes = INSTANT

💡 Tip: Virtualization itu seperti layar kaca — kamu hanya bisa lihat apa yang ada di depan mata, tapi di belakangnya masih ada ribuan item. react-window hanya bikin DOM untuk yang terlihat, sisanya virtual.

4. React DevTools Profiler & Best Practices

React DevTools Profiler membantu mengidentifikasi komponen yang lambat. Gabungkan dengan best practices untuk aplikasi yang optimal.

// Best Practices Checklist:
//
// ✅ 1. Gunakan key yang stabil (bukan index array)
// ❌ <li key={index}> — error-prone saat reorder
// ✅ <li key={item.id}> — stabil
//
// ✅ 2. Hindari inline object/array creation di JSX
// ❌ style={{ color: 'red' }} — object baru setiap render
// ✅ const style = useMemo(() => ({ color: 'red' }), [])
//
// ✅ 3. Hindari function creation di JSX (untuk child yang memo)
// ❌ onClick={() => doSomething()} — fungsi baru setiap render
// ✅ onClick={handleClick} — referensi stabil
//
// ✅ 4. Lazy load heavy components
// ✅ 5. Use production build
// ✅ 6. Measure before optimizing (Profiler!)
// ✅ 7. Avoid unnecessary state updates
// ✅ 8. Keep component tree flat

⚠️ Warning: Golden rule: 'Measure first, optimize second'. Jangan optimasi berdasarkan feeling — buka Profiler, lihat flame chart, cari komponen yang lambat, baru optimasi yang benar-benar perlu.
