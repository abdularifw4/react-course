React Hooks Advanced

1. useReducer — State Management Kompleks

Ketika state menjadi terlalu kompleks untuk useState, useReducer adalah solusinya. Pattern ini mirip Redux tapi built-in di React. useReducer menerima reducer function dan initial state, lalu mengembalikan state dan dispatch function.

import { useReducer } from 'react';

// Reducer: fungsi murni yang tentukan state baru
function todoReducer(state, action) {
  switch (action.type) {
    case 'ADD':
      return [...state, { id: Date.now(), text: action.payload, done: false }];
    case 'TOGGLE':
      return state.map(t =>
        t.id === action.payload ? { ...t, done: !t.done } : t
      );
    case 'DELETE':
      return state.filter(t => t.id !== action.payload);
    default:
      return state;
  }
}

function TodoApp() {
  const [todos, dispatch] = useReducer(todoReducer, []);
  const [input, setInput] = useState('');

  const addTodo = () => {
    if (input.trim()) {
      dispatch({ type: 'ADD', payload: input });
      setInput('');
    }
  };

  return (
    <div>
      <input value={input} onChange={e => setInput(e.target.value)} />
      <button onClick={addTodo}>Tambah</button>
      {todos.map(t => (
        <div key={t.id}>
          <span style={{textDecoration: t.done ? 'line-through' : 'none'}}>
            {t.text}
          </span>
          <button onClick={() => dispatch({type:'TOGGLE',payload:t.id})}>Toggle</button>
          <button onClick={() => dispatch({type:'DELETE',payload:t.id})}>Hapus</button>
        </div>
      ))}
    </div>
  );
}

💡 Tip: useReducer itu seperti mesin tik: kamu kirim 'perintah' (dispatch), dan mesin menentukan apa yang terjadi (reducer). Lebih predictable daripada setState karena semua logic state ada di satu tempat.

2. Custom Hooks — Reusability Logic

Custom hooks memungkinkan kamu mengekstrak dan mereuse logic komponen. Cukup buat fungsi yang dimulai dengan 'use', dan kamu bisa menggunakan hooks lain di dalamnya. Ini adalah cara paling powerful untuk share logic antar komponen.

// Custom hook: useLocalStorage
function useLocalStorage(key, initialValue) {
  const [value, setValue] = useState(() => {
    const saved = localStorage.getItem(key);
    return saved ? JSON.parse(saved) : initialValue;
  });

  useEffect(() => {
    localStorage.setItem(key, JSON.stringify(value));
  }, [key, value]);

  return [value, setValue];
}

// Custom hook: useWindowSize
function useWindowSize() {
  const [size, setSize] = useState({ width: 0, height: 0 });
  useEffect(() => {
    const handler = () => setSize({
      width: window.innerWidth,
      height: window.innerHeight
    });
    handler();
    window.addEventListener('resize', handler);
    return () => window.removeEventListener('resize', handler);
  }, []);
  return size;
}

// Custom hook: useDebounce
function useDebounce(value, delay) {
  const [debounced, setDebounced] = useState(value);
  useEffect(() => {
    const timer = setTimeout(() => setDebounced(value), delay);
    return () => clearTimeout(timer);
  }, [value, delay]);
  return debounced;
}

// Pakai di komponen
function SearchBar() {
  const [query, setQuery] = useState('');
  const debouncedQuery = useDebounce(query, 500);
  // Fetch dengan debouncedQuery
}

💡 Tip: Custom hooks itu seperti resep masakan. Kamu campur beberapa bahan (hooks bawaan) menjadi satu masakan baru yang bisa dipakai berkali-kali di berbagai komponen.

3. Rules of Hooks — Aturan yang Wajib Dipatuhi

React memiliki dua aturan wajib tentang hooks. Melanggar aturan ini akan menyebabkan bug yang sulit dilacak. ESLint plugin eslint-plugin-react-hooks bisa menangkap pelanggaran ini secara otomatis.

// ❌ ATURAN 1: Jangan panggil hooks di dalam condition/loop
function BadComponent({ showCounter }) {
  if (showCounter) {
    const [count, setCount] = useState(0); // ERROR!
  }
  const [theme, setTheme] = useState('dark'); // ERROR - hook ke-2 terkadang
}

// ✅ BENAR: Selalu di top level
function GoodComponent({ showCounter }) {
  const [count, setCount] = useState(0);
  const [theme, setTheme] = useState('dark');
  if (!showCounter) return null; // Condition setelah semua hooks
}

// ❌ ATURAN 2: Jangan panggil hooks dari fungsi JavaScript biasa
function badHelper() {
  const [value, setValue] = useState(''); // ERROR!
}

// ✅ BENAR: Hooks hanya di React components atau custom hooks
function useMyHook() { // Nama dimulai 'use' -> custom hook
  const [value, setValue] = useState('');
  return [value, setValue];
}

⚠️ Warning: Aturan ini ada karena React mengandalkan urutan pemanggilan hooks untuk mengasosiasikan state dengan komponen yang benar. Kalau urutannya berubah, React bingung!

4. useTransition & useDeferredValue

useTransition memungkinkan kamu menandai update state tertentu sebagai 'non-urgent', sehingga UI tetap responsif saat ada update berat. useDeferredValue mirip tapi untuk values yang sudah ada.

import { useState, useTransition, useDeferredValue } from 'react';

// useTransition: menandai update sebagai non-urgent
function SearchPage() {
  const [query, setQuery] = useState('');
  const [results, setResults] = useState([]);
  const [isPending, startTransition] = useTransition();

  const handleChange = (e) => {
    const value = e.target.value;
    setQuery(value); // Urgent: input langsung update
    startTransition(() => {
      setResults(filterResults(value)); // Non-urgent: bisa ditunda
    });
  };

  return (
    <div>
      <input value={query} onChange={handleChange} />
      {isPending ? <Spinner /> : <ResultList results={results} />}
    </div>
  );
}

// useDeferredValue: defer value yang sudah ada
function FilterList({ items, filter }) {
  const deferredFilter = useDeferredValue(filter);
  const filtered = items.filter(item =>
    item.name.includes(deferredFilter)
  );
  return <List items={filtered} />;
}

💡 Tip: useTransition itu seperti antrian VIP di restoran. Input user = VIP (langsung dilayani), pencarian hasil = antrian biasa (bisa menunggu). Ini mencegah UI freeze saat ada proses berat.

5. useSyncExternalStore

useSyncExternalStore memungkinkan komponen React subscribe ke external data source (di luar React) secara aman dari race conditions. Digunakan oleh library state management modern.

import { useSyncExternalStore } from 'react';

// Subscribe ke browser online status
function OnlineStatus() {
  const isOnline = useSyncExternalStore(
    (callback) => {
      window.addEventListener('online', callback);
      window.addEventListener('offline', callback);
      return () => {
        window.removeEventListener('online', callback);
        window.removeEventListener('offline', callback);
      };
    },
    () => navigator.onLine,      // Client snapshot
    () => true                    // Server snapshot (SSR)
  );

  return <span>{isOnline ? '🟢 Online' : '🔴 Offline'}</span>;
}

// Subscribe ke media query
function useMediaQuery(query) {
  return useSyncExternalStore(
    (callback) => {
      const mql = window.matchMedia(query);
      mql.addEventListener('change', callback);
      return () => mql.removeEventListener('change', callback);
    },
    () => window.matchMedia(query).matches,
    () => false
  );
}

💡 Tip: useSyncExternalStore adalah jembatan antara dunia React dan dunia non-React. Seperti charger universal — bisa nyambung ke mana saja.
