React & TypeScript

1. Setup React + TypeScript

TypeScript menambahkan type safety ke React, membantu menangkap bug sebelum runtime. Setup-nya mudah: gunakan template resmi create-react-app atau Vite.

// Install project React + TypeScript
// npm create vite@latest my-app -- --template react-ts

// Atau dengan CRA:
// npx create-react-app my-app --template typescript

// Struktur file:
// src/
//   components/
//     Button.tsx
//     Card.tsx
//   types/
//     index.ts
//   App.tsx
//   main.tsx

// Konfigurasi tsconfig.json penting:
// "strict": true — aktifkan semua type checking ketat
// "jsx": "react-jsx" — tidak perlu import React di setiap file

💡 Tip: Mulai dari 'strict: true' sejak awal. Memang lebih ribet di awal, tapi ini mencegah ratusan bug di produksi. Better safe than sorry!

2. Typing Props & State

Definisikan tipe props menggunakan interface atau type alias. Ini adalah fondasi TypeScript di React — setiap komponen harus punya tipe props yang jelas.

// Interface untuk props
interface UserCardProps {
  name: string;
  email: string;
  age?: number; // Optional
  role: 'admin' | 'user' | 'guest'; // Union type
  onEdit: (id: string) => void;
  children: React.ReactNode;
}

function UserCard({ name, email, age, role, onEdit, children }: UserCardProps) {
  return (
    <div>
      <h3>{name}</h3>
      <p>{email}</p>
      {age && <p>Umur: {age}</p>}
      <span>{role}</span>
      <button onClick={() => onEdit('123')}>Edit</button>
      {children}
    </div>
  );
}

// Typing state
interface FormState {
  username: string;
  password: string;
  errors: Record<string, string>;
  isLoading: boolean;
}

function LoginForm() {
  const [state, setState] = useState<FormState>({
    username: '',
    password: '',
    errors: {},
    isLoading: false
  });
}

💡 Tip: Bayangkan type itu seperti kontrak. Interface = 'komponen ini janji menerima name (string) dan role (admin/user/guest)'. Kalau langgar kontrak, TypeScript langsung nolong di coding time, bukan di produksi.

3. Generics di Komponen

Generics memungkinkan komponen menerima berbagai tipe data tanpa kehilangan type safety. Sangat berguna untuk reusable components.

// Generic List component
interface ListProps<T> {
  items: T[];
  renderItem: (item: T) => React.ReactNode;
  keyExtractor: (item: T) => string;
  onItemSelect?: (item: T) => void;
}

function List<T>({ items, renderItem, keyExtractor, onItemSelect }: ListProps<T>) {
  return (
    <ul>
      {items.map(item => (
        <li key={keyExtractor(item)} onClick={() => onItemSelect?.(item)}>
          {renderItem(item)}
        </li>
      ))}
    </ul>
  );
}

// Pakai dengan tipe User
interface User { id: string; name: string; }
const users: User[] = [{ id: '1', name: 'Arif' }];

<List<User>
  items={users}
  keyExtractor={u => u.id}
  renderItem={u => <span>{u.name}</span>}
  onItemSelect={u => console.log(u)}
/>

// Pakai dengan tipe Product
interface Product { sku: string; title: string; price: number; }
<List<Product>
  items={products}
  keyExtractor={p => p.sku}
  renderItem={p => <span>{p.title} - Rp{p.price}</span>}
/>

💡 Tip: Generics itu seperti template surat. Format suratnya sama, tapi isi-nya bisa beda-beda tergantung kebutuhan. T tetap type-safe karena TypeScript tahu T-nya apa.

4. Utility Types: Partial, Pick, Omit, Record

TypeScript punya utility types bawaan yang sangat berguna untuk memanipulasi tipe tanpa menulis ulang.

interface User {
  id: string;
  name: string;
  email: string;
  age: number;
  role: 'admin' | 'user';
}

// Partial: semua prop jadi optional
function updateUser(id: string, data: Partial<User>) {
  // data bisa { name: 'baru' } saja, tidak perlu semua field
}

// Pick: pilih beberapa prop saja
type UserPreview = Pick<User, 'id' | 'name'>;
// { id: string; name: string }

// Omit: exclude beberapa prop
type CreateUser = Omit<User, 'id'>;
// { name: string; email: string; age: number; role: 'admin'|'user' }

// Record: bikin tipe object dengan key dan value tertentu
type ThemeColors = Record<string, string>;
const colors: ThemeColors = {
  primary: '#27d890',
  secondary: '#111827',
  danger: '#ef4444'
};

// Required: semua prop jadi wajib
type RequiredUser = Required<Partial<User>>;

// Readonly: tidak bisa diubah
type FrozenUser = Readonly<User>;

💡 Tip: Utility types itu seperti alat multitool — Partial, Pick, Omit, Record masing-masing punya fungsi spesifik untuk memanipulasi tipe tanpa copy-paste.
