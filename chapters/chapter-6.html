01

Setup React + TypeScript

Persiapan · 5 menit baca

TypeScript adalah superset JavaScript yang menambahkan static typing. Saat digunakan bersama React, TypeScript membantu mendeteksi error lebih awal, memberikan autocompletion lebih baik di IDE, dan membuat kode lebih mudah dipahami.

Cara paling mudah memulai proyek React + TypeScript adalah dengan Vite. Vite mendukung template TypeScript secara native — cukup tambahkan flag --template react-ts saat membuat proyek.

File komponen React + TypeScript menggunakan ekstensi .tsx, file utility biasa menggunakan .ts. Vite otomatis mengkonfigurasi tsconfig.json.

Terminal — Setup
npm create vite@latest my-app -- --template react-ts
cd my-app && npm install && npm run dev

💡 Analogi: TypeScript seperti spell checker untuk kode. Seperti spell checker menangkap typo sebelum kamu mengirim email, TypeScript menangkap error type sebelum kamu deploy aplikasi.

02

Typing Props & State

Fundamental · 8 menit baca

Cara paling umum mendefinisikan tipe props adalah interface. Interface bisa di-extend dan lebih familiar bagi developer React. Untuk state, gunakan generic type parameter pada useState.

Gunakan React.FC opsional — banyak developer modern lebih suka mendefinisikan komponen sebagai fungsi biasa dengan tipe props eksplisit, karena React.FC memiliki beberapa kelemahan (seperti implisit children).

Untuk children, gunakan React.ReactNode sebagai tipe. Untuk event handler, gunakan tipe event yang sesuai seperti React.ChangeEvent atau React.MouseEvent.

Typing Props & State
// Mendefinisikan interface props
interface UserCardProps {
  name: string;
  age: number;
  email?: string;  // optional
  role: 'admin' | 'user' | 'editor';  // union type
  children?: React.ReactNode;
}

// Komponen dengan typed props
function UserCard({ name, age, email, role, children }: UserCardProps) {
  const [isExpanded, setIsExpanded] = useState<boolean>(false);
  return (
    <div>
      <h2>{name} ({age})</h2>
      <p>Role: {role}</p>
      {email && <p>{email}</p>}
      {children}
      <button onClick={() => setIsExpanded(!isExpanded)}>
        {isExpanded ? 'Tutup' : 'Buka'}
      </button>
    </div>
  );
}

// Penggunaan — TypeScript akan memvalidasi props
function App() {
  return (
    <UserCard name="Budi" age={25} role="admin">
      <p>Konten tambahan</p>
    </UserCard>
  );
}

💡 Analogi: Interface props seperti kontrak kerja. Keduanya menentukan apa yang wajib disediakan (required props), apa yang opsional (optional props), dan apa saja nilai yang diperbolehkan (union types).

03

Generics, Events, Refs & Hooks

Menengah · 10 menit baca

Generics memungkinkan komponen menerima tipe sebagai parameter. Ini sangat berguna untuk komponen reusable yang bekerja dengan berbagai tipe data, seperti list, form, atau data table.

Untuk Events, React menyediakan tipe event spesifik: ChangeEvent<HTMLInputElement>, FormEvent, MouseEvent, dll. Untuk refs, gunakan useRef<HTMLDivElement>(null).

Custom Hooks juga bisa di-type dengan Generics. Gunakan parameter generic pada fungsi hook untuk membuat hook yang flexible namun type-safe.

Generics & Typing
// Generic Component — List yang bisa menampilkan tipe apapun
interface ListProps<T> {
  items: T[];
  renderItem: (item: T) => React.ReactNode;
  keyExtractor: (item: T) => string;
}

function List<T>({ items, renderItem, keyExtractor }: ListProps<T>) {
  return <ul>{items.map(item => (
    <li key={keyExtractor(item)}>{renderItem(item)}</li>
  ))}</ul>;
}

// Typing Events
function SearchForm() {
  const handleInput = (e: React.ChangeEvent<HTMLInputElement>) => {
    console.log(e.target.value);
  };
  const handleSubmit = (e: React.FormEvent) => {
    e.preventDefault();
  };
  return (
    <form onSubmit={handleSubmit}>
      <input onChange={handleInput} />
    </form>
  );
}

// Typing Refs
function FocusInput() {
  const inputRef = useRef<HTMLInputElement>(null);
  useEffect(() => { inputRef.current?.focus(); }, []);
  return <input ref={inputRef} />;
}

// Generic Custom Hook
function useLocalStorage<T>(key: string, initial: T) {
  const [value, setValue] = useState<T>(() => {
    const saved = localStorage.getItem(key);
    return saved ? JSON.parse(saved) : initial;
  });
  useEffect(() => {
    localStorage.setItem(key, JSON.stringify(value));
  }, [key, value]);
  return [value, setValue] as const;
}

💡 Analogi: Generics seperti template surat. Format suratnya sama, tapi kamu bisa mengisi dengan konten berbeda (string, number, object) sesuai kebutuhan tanpa membuat template baru.

04

Utility Types & Discriminated Unions

Menengah · 8 menit baca

TypeScript menyediakan Utility Types bawaan yang sangat powerful: Partial (semua prop jadi optional), Pick (pilih prop tertentu), Omit (hapus prop tertentu), dan Record (buat objek dari key-value).

Discriminated Unions adalah pattern di mana union type menggunakan prop discriminator (biasanya type atau status) untuk membedakan antar anggota union. Ini memungkinkan TypeScript melakukan type narrowing otomatis.

Utility Types sangat berguna saat kamu perlu memanipulasi tipe tanpa copy-paste. Discriminated Unions sempurna untuk merepresentasikan state yang berbeda (loading, success, error) dalam satu type yang aman.

Utility Types & Discriminated Unions
interface User { id: number; name: string; email: string; age: number; }

// Partial — semua prop jadi optional
type PartialUser = Partial<User>;

// Pick — ambil prop tertentu saja
type UserPreview = Pick<User, 'id' | 'name'>;

// Omit — hapus prop tertentu
type UserWithoutAge = Omit<User, 'age'>;

// Record — buat tipe objek dari key-value
type RolePermissions = Record<string, string[]>;
const perms: RolePermissions = {
  admin: ['read', 'write', 'delete'],
  user: ['read'],
};

// Discriminated Unions — state management yang type-safe
type AsyncState<T> =
  | { status: 'idle' }
  | { status: 'loading' }
  | { status: 'success'; data: T }
  | { status: 'error'; error: Error };

function UserProfile() {
  const [state, setState] = useState<AsyncState<User>>({
    status: 'idle'
  });

  // TypeScript tahu data hanya ada saat status === 'success'
  if (state.status === 'success') {
    return <p>{state.data.name}</p>;  // state.data = User ✓
  }
  if (state.status === 'error') {
    return <p>{state.error.message}</p>;  // state.error = Error ✓
  }
  return <p>Loading...</p>;
}

💡 Analogi: Utility Types seperti alat multitool di Swiss Army Knife. Partial, Pick, Omit, Record masing-masing punya fungsi spesialis tapi semuanya berguna untuk memanipulasi tipe tanpa menulis ulang definisi type yang sama berulang kali.
